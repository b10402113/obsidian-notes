# 微服務部署：雲端 VM、專屬主機與實例

## 📝 課程概述

本單元是 Chapter 8（微服務部署與生產環境）的第一堂課。我們將探討在雲端環境中部署 Microservice 的三種基礎設施選項：多租戶（Multi-tenant）雲端虛擬機、單租戶（Single-tenant）專屬實例，以及專屬主機（Dedicated Host）。理解這三種選項的價格、安全性與效能特性，是做出正確部署決策的前提。

---

## 核心觀念與實作解析

### 雲端虛擬機的基礎運作模型

雲端 VM 是以軟體（**Hypervisor**）在實體硬體上模擬出多個隔離的虛擬電腦：

- 每個 VM 有自己的 OS、CPU、記憶體、網路介面與儲存
- Hypervisor 負責將虛擬資源映射到實體資源，並根據付費配置動態分配
- 同一台實體伺服器上可以同時運行：同一 Microservice 的多個 Instance、不同 Microservice、甚至不同客戶的 VM

這個特性稱為 **Multi-tenancy（多租戶）**，也是雲端 VM 能提供低廉價格的核心原因——雲端供應商可以極高效地分割硬體資源，實現高利用率。

---

### 多租戶 VM 的定價優勢

雲端 VM 採用 **Pay-per-use（按用量付費）** 的定價模型：

- 只為實際使用的時段付費
- 費率由配置的 CPU、Memory 與網路頻寬決定
- 相較於自建機房，無需負擔硬體採購、維護與機房成本

---

### 多租戶 VM 的安全風險：Hypervisor 漏洞

**理論上**，每個 VM 都是完全隔離的，因此不同組織的 VM 運行在同一台實體伺服器上不應該互相影響。

但現實世界中：
- Hypervisor 是人寫的軟體，存在被攻擊的可能性
- 如果另一個組織的 VM 配置不當，駭客可能透過該 VM 突破 Hypervisor 層，進而影響到同一實體主機上的其他 VM

> **Compliance 限制**：銀行、醫療、國防等產業的法規，往往**完全不允許**將敏感資料放在多租戶環境中。

---

### 多租戶 VM 的效能風險：Noisy Neighbor（吵鬧的鄰居）

**另一個多租戶 VM 的潛在問題**——並非所有實體資源都能被精確地分配給 VM：

- **CPU Core**：可以輕易分割（16 Core → 各 8 Core）
- **網路頻寬、Internal Bus、Storage Access**：難以精確配額
- Hypervisor 本身也會消耗 CPU 與記憶體

因此，當同一台實體伺服器上的某個 VM 運行高度密集的工作負載時，**理論上會對同主機上的其他 VM 造成效能影響**。

> 對延遲極度敏感的場景（高頻交易、遊戲、影像串流），多租戶 VM 並不是最佳選擇。

---

### 選項二：Single-Tenant Dedicated Instances（單租戶專屬實例）

如果多租戶環境無法滿足安全合規需求，可以要求雲端供應商將 VM 部署在**僅屬於自己組織的實體伺服器**上：

- 主機上只有自己組織的 Microservice Instance 與 Database
- 滿足「不可與其他公司共享基礎設施」的合規要求
- 仍保有 VM 的抽象層便利性

**代價**：雲端供應商無法像多租戶那樣高效地利用硬體，因此收費較高。

---

### 選項三：Dedicated Hosts（專屬主機）

最高等級的隔離——直接租用**整台實體主機**：

- 對硬體資源有直接存取能力，**完全消除 Noisy Neighbor 效應**
- **消除 Hypervisor 的虛擬化開銷**
- 適合對效能與隔離有最嚴格要求的場景

**代價**：成本比 Dedicated Instances 還要高，是三種選項中最昂貴的。

---

### 三種部署選項的完整比較

| 維度 | Multi-tenant VM | Dedicated Instance | Dedicated Host |
|---|---|---|---|
| **定價** | 最低 | 中等 | 最高 |
| **安全性** | 低（Hypervisor 漏洞風險） | 高（單租戶隔離） | 最高（直接控制硬體） |
| **效能** | 中（有 Noisy Neighbor 風險） | 高 | 最高（無虛擬化開銷） |
| **合規適用性** | 銀行、醫療、國防不可用 | 多數合規需求可滿足 | 所有最嚴格合規需求 |
| **Noisy Neighbor** | 存在理論風險 | 消除 | 消除 |

---

## 💡 重點摘要

- **雲端 VM 的 Multi-tenancy 特性是低價格的來源，但同時也是安全與效能風險的根源。**
- **Hypervisor 漏洞在理論上可能讓攻擊者穿透同一實體主機上的其他租戶 VM——這是某些產業法規完全禁止多租戶部署的原因。**
- **Noisy Neighbor 問題源於網路頻寬、Internal Bus 等實體資源無法被精確分割，高效能需求場景應避免多租戶 VM。**
- **Dedicated Instance 在安全性與成本之間取得平衡；Dedicated Host 適合對隔離性與效能有最高要求的場景。**
- **部署選項沒有絕對的好壞——必須根據安全性合規需求、效能敏感性與預算来做權衡取捨。**
