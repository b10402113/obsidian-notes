# 微服務部署：雲端虛擬機、專屬主機與實例

## 📝 課程概述

本單元是 Chapter 8 的第一堂課，聚焦於 Microservice 部署的基礎設施選型。我們將從最常見的 Cloud VM（多租戶虛擬機）出發，分析其定價優勢與安全/效能的代價；接著介紹兩種逐步加強隔離的替代方案——Single Tenant Dedicated Instances 與 Dedicated Hosts，幫助學員根據產業合規需求與預算，針對不同情境做出正確的部署決策。

---

## 核心觀念與實作解析

### Cloud VM（多租戶虛擬機）：最普遍的部署選項

#### 什麼是 Cloud VM？

Cloud VM 是一種運行在真實實體機器上的隔離虛擬環境。底層的 **Hypervisor（虛擬機器監視器）**負責將實體硬體（CPU、記憶體、網路卡、儲存）虛擬化，並分配給每台 VM。

一台實體伺服器可以拆分為多台 VM，每台 VM：
- 各自擁有獨立的 OS 與虛擬資源
- 可能運行同一個 Microservice 的不同 Instance
- 也可能運行不同組織的服務

這種特性稱為 **Multi-tenancy（多租戶）**。

#### Cloud VM 的定價優勢

Cloud VM 的標準定價模型是 **Pay-per-use（按使用量付費）**：
- 只為實際租用 VM 的時段付費
- 費率取決於你所申請的 CPU、記憶體、網路頻寬配置

> 這使得 Cloud VM 成為部署 Microservice 最經濟實惠的起點。

---

### 多租戶環境的兩大成代價

#### 代價一：安全風險

Hypervisor 軟體理論上能完全隔離不同 VM——但軟體由人維護，人會犯錯。現實中曾有案例：

- VM A 的工程師配置不當 → Hacker 入侵 VM A
- 因 Hypervisor 隔離失效 → Hacker 滲透到同一實體主機上的 VM B
- VM B 的組織資料因此被竊取

> **金融、醫療、國防相關的服務往往受到法規約束，完全無法接受這種哪怕機率極低的安全風險。**

#### 代價二：Noisy Neighbor（吵鬧的鄰居）

某些硬體資源**難以精確分割**：

- **CPU 核心**可以均分（16 核 → 每 VM 8 核）
- **網路頻寬**與**儲存匯流排**難以精準配給
- Hypervisor 本身也會消耗 CPU 與記憶體

結果：高流量 VM 的工作負擔會**實質影響**同實體主機上其他 VM 的延遲。

> **對延遲高度敏感的服務（高頻交易、遊戲、影像串流），多租戶 VM 不是理想選擇。**

---

### Single Tenant Dedicated Instances（單租戶專屬實例）

若你的產業**不允許與其他公司共享基礎設施**，可以向雲端供應商申請專屬實例：

- VM 只會部署在你所屬組織的帳戶下
- 同一實體主機上只有你自己的 Microservice Instance 與 Database
- 消除了 Multi-tenancy 安全風險

> 代價：定價比多租戶 VM 更高（因為雲端供應商無法像多租戶那樣高效地利用硬體）。

---

### Dedicated Hosts（專屬主機）

比 Dedicated Instances 更進一步——**直接租用整台實體主機**：

- 完全消除 Noisy Neighbor 效應
- 減少 Hypervisor 虛擬化的額外損耗
- 對硬體資源有直接存取能力

> 代價：**最昂貴的選項**。通常只適合對延遲、效能與隔離有極高要求的組織。

---

### 三層部署選項的權衡總覽

| 維度 | 多租戶 VM | Dedicated Instances | Dedicated Hosts |
|---|---|---|---|
| **定價** | 最便宜 | 中等 | 最昂貴 |
| **安全性** | 有潛在風險（Multi-tenancy） | 僅自己組織的 VM | 完全隔離 |
| **效能穩定性** | 可能受 Noisy Neighbor 影響 | 大幅改善 | 完全消除 Noisy Neighbor |
| **適用場景** | 多數一般 Web 服務 | 金融、醫療等合規需求 | 高效能/低延遲需求 |

---

## 💡 重點摘要

- **Cloud VM 透過 Multi-tenancy 實現高效硬體利用與靈活定價，是部署 Microservice 最經濟的起點。**
- **多租戶 VM 的兩大成代價：Hypervisor 隔離漏洞導致的安全風險，以及 Noisy Neighbor 導致的效能不穩定。**
- **Single Tenant Dedicated Instances 消除跨組織的安全共享問題，適合有合規要求的產業。**
- **Dedicated Hosts 提供最完整的效能與安全隔離，代價是最高的基礎設施成本。**
- **選擇部署選項時，應以產業合規需求、延遲敏感度與預算為依據，而非一味追求最高規格。**
