# 容器映像檔的內容解析

## 📝 課程概述

本單元是整個 Chapter 5 的開場，老師帶領我們深入理解 Container Image 的本質：它包含什麼、不包含什麼，以及如何判斷 Docker Hub 上的 Image 品質。這些觀念是後續建立自訂 Image 的基礎。

---

## 核心觀念與實作解析

### Container Image 裡面有什麼？

老師用最直觀的方式定義 Image：

> **Image = 應用程式二進位檔 + 依賴項目 + 執行相關的 Metadata**

官方定義則更為正式：「Image 是 root filesystem 變更的有序集合，以及對應的執行參數。」

#### Image 不包含什麼？

這是 Container 與 VM 的關鍵差異：

| Container Image 包含 | Container Image 不包含 |
|---------------------|----------------------|
| 應用程式 binaries | Kernel |
| 應用程式依賴的 libraries | Kernel modules（如驅動程式） |
| Metadata | 完整的 OS |

**為什麼？** 因為 Host 已經提供 Kernel。Container 不需要啟動完整的作業系統，它只是啟動一個應用程式。

#### Image 大小的差異

Image 可以非常小，也可以很大：

- **極小 Image**：例如 Go 語言的 static binary，可能只有單一檔案
- **大型 Image**：例如包含 Ubuntu、Apache、PHP、原始碼與各種模組，可能數 GB

---

### Docker Hub 導覽

Docker Hub 是 Docker 官方的 Image Registry，類似 Container 世界的 `apt` 套件管理器。

#### 官方 Image 的辨識方式

在 Docker Hub 搜尋時，如何辨識官方 Image？

**特徵一：名稱沒有斜線**
- 官方：`nginx`、`mysql`、`python`
- 非官方：`bretfisher/nginx`、`mysql/mysql-server`

**特徵二：標示 "Official"**
- 官方 Image 會在名稱下方顯示 "Docker Official Image" 標籤

**官方 Image 的優勢：**
- Docker, Inc. 有專門團隊維護
- 品質文件與最佳實踐
- 與軟體官方團隊合作確保正確性

---

### Tag 與版本管理

#### Tag 的概念

Image 本身沒有「名稱」，我們透過 **Repository + Tag** 來識別它。

```
docker pull nginx:1.11.9
           │       │
           │       └── Tag（版本標籤）
           └────────── Repository 名稱
```

#### `latest` Tag 的陷阱

`latest` 是預設 Tag，但它有一些需要注意的地方：

> **`latest` 不保證是最新版本**，它只是一個預設標籤名稱。你完全可以把舊軟體標記為 `latest`。

**最佳實踐：**
- 開發/測試環境：可以使用 `latest` 方便快速
- **生產環境：一定要指定明確版本**，避免自動更新造成不可預期的問題

---

### Base Image 與 Alpine

#### Alpine vs Debian

老師特別介紹 Alpine Linux 這個輕量級發行版：

| Base Image | 大小 | 特點 |
|-----------|------|------|
| **Alpine** | ~5MB | 極小、安全、適合生產環境 |
| **Debian (jessie)** | ~100MB+ | 功能完整、相容性高 |

**實際比較：**
- `nginx:latest`（Debian base）：182MB
- `nginx:alpine`：54MB

> Alpine 版本的 Image 大小通常只有 Debian 版本的 1/3 到 1/4。

### 如何判斷 Image 品質？

當考慮使用非官方 Image 時，可以參考以下指標：

1. **Pulls 數量** — 下載次數越多，通常表示可信度高
2. **Stars 數量** — 社群認可程度
3. **是否有 GitHub 連結** — 能檢視原始碼與 Dockerfile
4. **文件品質** — 說明是否清楚

> **安全建議：** 下載任何 Image 後，都應檢查其 Dockerfile 與原始碼，確保沒有安全問題。

---

## 💡 重點摘要

- **Container Image 只包含應用程式與其依賴，不包含 Kernel 與驅動程式，這是與 VM 的根本差異。**
- **官方 Image 名稱沒有斜線，由 Docker 官方團隊維護，文件品質與安全性較有保障。**
- **`latest` Tag 只是預設標籤，不保證是最新版本；生產環境應明確指定版本號。**
- **Alpine 是極小的 Linux 發行版（~5MB），使用 Alpine base 的 Image 通常更小。**
- **判斷非官方 Image 品質可參考 Pulls、Stars、GitHub 連結與文件完整性。**
