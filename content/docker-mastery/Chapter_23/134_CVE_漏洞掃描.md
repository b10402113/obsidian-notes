# CVE 漏洞掃描

## 📝 課程概述

本單元介紹如何在 GitHub Actions 中整合 CVE（Container Vulnerability Enumeration）掃描。講師使用 Trivy 工具示範從「資訊性掃描」到「阻斷性掃描」的導入過程，並說明如何將掃描結果上傳到 GitHub Security。

---

## 核心觀念與實作解析

### 為什麼要掃描 Image 漏洞？

Container Image 通常基於基礎映像檔（如 `node:18-alpine`），並安裝了各種相依套件。這些套件可能存在已知的安全漏洞（CVE）。在推送 Image 到生產環境前，應該：

1. **識別漏洞**：知道有哪些 CVE
2. **評估風險**：根據嚴重程度排序
3. **修補或接受**：更新套件或記錄接受風險的理由

---

### Trivy：免費開源的漏洞掃描器

**Trivy** 由 Aqua Security 開發，是目前最受歡迎的開源漏洞掃描器之一。

#### 非阻斷性掃描（資訊性）

```yaml
- name: Run Trivy vulnerability scanner
  uses: aquasecurity/trivy-action@master
  with:
    image-ref: 'user/image:${{ github.run_id }}'
    format: 'table'
    exit-code: '0'
```

- `exit-code: '0'`：即使發現漏洞，Workflow 也會繼續
- `format: 'table'`：以表格格式輸出到 Log

#### GitHub Run ID 作為標籤

```yaml
image-ref: 'user/image:${{ github.run_id }}'
```

講師習慣使用 `github.run_id` 作為測試 Image 的標籤：

- 每次 Workflow 執行都有唯一的數字 ID
- 方便辨識、複製貼上
- 避免多人同時推送 PR 時混淆

---

### 阻斷性掃描（正式導入）

當團隊準備好後，可以將掃描升級為「阻斷性」：

```yaml
permissions:
  security-events: write

- name: Run Trivy vulnerability scanner
  uses: aquasecurity/trivy-action@master
  with:
    image-ref: 'user/image:${{ github.run_id }}'
    format: 'sarif'
    output: 'trivy-results.sarif'
    severity: 'HIGH,CRITICAL'
    exit-code: '1'
    ignore-unfixed: true

- name: Upload Trivy scan results to GitHub Security
  uses: github/codeql-action/upload-sarif@v2
  with:
    sarif_file: 'trivy-results.sarif'
```

#### 關鍵參數說明

| 參數 | 說明 |
| --- | --- |
| `severity: 'HIGH,CRITICAL'` | 只報告高危與嚴重漏洞 |
| `exit-code: '1'` | 發現漏洞時中斷 Workflow |
| `ignore-unfixed: true` | 忽略尚未有修補方案的漏洞 |
| `format: 'sarif'` | 輸出 SARIF 格式，可上傳到 GitHub Security |

---

### GitHub Security 整合

上傳掃描結果後，可以在 GitHub Repo 的 **Security** 分頁查看：

- 漏洞列表與嚴重程度
- 受影響的套件版本
- 修補建議

> **最佳實踐**：先從非阻斷性掃描開始，了解團隊 Image 的漏洞概況，逐步改善後再啟用阻斷性掃描。一開始就用嚴格的阻斷性掃描可能會讓團隊無法推進。

---

### 權限設定（安全最佳實踐）

```yaml
permissions:
  contents: read
  security-events: write
```

講師建議在 Workflow 中明確宣告權限，遵循 **最小權限原則**：

- 預設只給 `read` 權限
- 只有需要的 Step 才給予 `write` 權限

---

## 💡 重點摘要

- **Trivy 是免費開源的漏洞掃描器，可掃描 Container Image 中的 CVE。**
- **建議從非阻斷性掃描開始，逐步改善漏洞清單後再啟用阻斷性掃描。**
- **掃描結果可上傳到 GitHub Security，方便追蹤與管理。**
- **使用 ignore-unfixed 選項忽略尚未有修補方案的漏洞。**
