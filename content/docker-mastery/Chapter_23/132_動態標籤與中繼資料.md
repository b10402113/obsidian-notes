# 動態標籤與中繼資料

## 📝 課程概述

本單元介紹 Docker Metadata Action，這個工具能根據 GitHub 事件自動產生智慧型的 Image 標籤與標籤（Labels）。透過宣告式的規則設定，你可以讓 PR、合併、Git Tag 等不同情境自動套用不同的標籤策略。

---

## 核心觀念與實作解析

### 為什麼需要動態標籤？

在實務中，不同階段會需要不同的標籤：

| 情境 | 標籤範例 |
| --- | --- |
| PR 建立 | `pr-15`、`sha-abc123` |
| 合併到 main | `main`、`latest` |
| Git Tag | `v1.2.3`、`1.2.3` |

手動撰寫腳本處理這些邏輯既繁瑣又容易出錯，Docker Metadata Action 可以自動完成。

---

### Docker Metadata Action 設定

#### 基本設定

```yaml
- name: Docker meta
  id: meta
  uses: docker/metadata-action@v4
  with:
    images: user/image
    tags: |
      type=ref,event=branch
      type=ref,event=pr
      type=semver,pattern={{version}}
      type=sha
```

#### 標籤規則類型

| 規則類型 | 說明 | 範例輸出 |
| --- | --- | --- |
| `type=ref,event=branch` | 分支名稱 | `main` |
| `type=ref,event=pr` | PR 編號 | `pr-15` |
| `type=semver,pattern={{version}}` | Semantic Version | `1.2.3` |
| `type=sha` | Git Commit SHA | `sha-abc1234` |

> **運作原理**：Action 會檢查當前的 GitHub 事件類型，自動套用對應的規則。一個 Image 可能同時擁有多個標籤。

---

### 使用產生的標籤

```yaml
- name: Build and push
  uses: docker/build-push-action@v4
  with:
    context: .
    push: true
    tags: ${{ steps.meta.outputs.tags }}
    labels: ${{ steps.meta.outputs.labels }}
```

> `steps.meta.outputs.tags` 會自動帶入上一個 Step 產生的所有標籤。

---

### 在 PR 中顯示標籤（加分功能）

為了避免在 Log 中翻找標籤，可以讓自動化在 PR 中留下評論：

```yaml
- name: Comment PR
  if: github.event_name == 'pull_request'
  uses: actions/github-script@v6
  with:
    script: |
      github.rest.issues.createComment({
        issue_number: context.issue.number,
        owner: context.repo.owner,
        repo: context.repo.repo,
        body: '🏷️ Docker image tags: ${{ steps.meta.outputs.tags }}'
      })
```

這樣每次 PR 更新時，都會自動在評論區顯示本次建置的 Image 標籤。

---

### 智慧推送策略

結合 Metadata Action 後，可以放心啟用 `push: true`：

```yaml
push: true
```

因為 Metadata Action 已經根據事件類型產生對應標籤：

- **PR 階段**：推送帶有 `pr-N` 標籤的 Image
- **合併階段**：推送帶有 `main`、`latest` 標籤的 Image

> **效益**：不需要手動判斷何時該推送、推送什麼標籤，全部自動化。

---

## 💡 重點摘要

- **Docker Metadata Action 根據 GitHub 事件自動產生智慧型標籤，支援分支名、PR 編號、Semantic Version、Git SHA。**
- **透過宣告式的標籤規則，避免手動撰寫複雜腳本。**
- **可以在 PR 中自動評論顯示 Image 標籤，方便開發者查找。**
- **結合 Metadata Action 後，可以放心啟用自動推送，由規則自動決定標籤。**
