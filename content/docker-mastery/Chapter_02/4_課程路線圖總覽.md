# Docker 課程路線圖總覽

## 📝 課程概述

本單元是整個 Docker Mastery 課程的導覽，Bret Fisher 老師將帶領我們快速瀏覽課程的學習路徑與各章節主題。我們會從 Docker 安裝開始，逐步深入 Container 基礎、Image 建立、Networking、Volumes，最後邁向 Orchestration（Swarm 與 Kubernetes），幫助學員建立清晰的學習方向。

---

## 核心觀念與實作解析

### 課程學習路徑總覽

老師將整個課程規劃為一條循序漸進的學習路徑，讓學員能夠從零開始，一步步建立 Docker 的完整知識體系。以下是各階段的重點內容：

---

### 第一階段：環境準備與安裝

在正式進入 Docker 之前，我們需要先完成以下準備工作：

1. **確認作業系統相容性** — 確保你的 OS 能夠順利運行 Docker
2. **下載課程 Repo** — 取得課程提供的範例程式碼與練習素材
3. **安裝 Docker 本體** — 根據你的環境選擇最適合的安裝方式

關於安裝細節，老師特別提到：

- **Docker Desktop** 是 Mac 與 Windows 10 的最佳選擇，安裝簡單且整合性高
- 如果你使用的是 Linux、舊版 Windows，或是想在雲端 Server 上運行 Docker，課程中也有對應的安裝指引

> 這些安裝細節之所以重要，是因為 Docker 的運作方式會因作業系統而異。理解這些差異能幫助你在未來除錯時更快定位問題。

---

### 第二階段：Docker 基礎核心

完成安裝後，我們將進入 Docker 的核心概念：

- **Container 是什麼？** — 理解 Container 的本質、裡面有什麼、裡面沒有什麼
- **如何運行 Container？** — 透過實際操作範例學會基本指令
- **Image 的建立** — Image 是 Container 的建構基石，我們將學習如何建立自己的 Image

> Container 與 Image 的關係可以想像成「物件」與「藍圖」：Image 是靜態的模板，Container 則是從 Image 啟動的運行實體。

---

### 第三階段：進階功能 — Networking 與 Volumes

在掌握基礎後，課程會深入兩個關鍵主題：

| 主題        | 說明                                     |
| ----------- | ---------------------------------------- |
| Networking  | Docker 的網路機制，讓 Container 間能夠互相通訊 |
| Volumes     | 資料持久化儲存，解決 Container 生命週期與資料保存的問題 |

這兩個主題是生產環境中不可或缺的知識，特別是當你需要建立多 Container 協作的架構時。

---

### 第四階段：Docker Compose — 本地開發利器

接著我們會學習 **Docker Compose**，這是一個隨 Docker 附帶的工具：

- **定位**：專為本地開發（Dev）與測試（Test）環境設計
- **優勢**：可以用一個 YAML 檔案定義多 Container 的應用架構

老師特別強調：

> 「我每天工作都會用到 Docker Compose。」這句話說明了這個工具在實務上的重要性。它能大幅簡化本地開發時的環境設定流程。

---

### 第五階段：Orchestration — 從單機到多機叢集

當你熟悉了 Docker 的基礎工具後，課程將進入 **Orchestration（編排）** 的領域：

> Orchestration 的核心概念是：讓你能在多台 Server 上運行 Container，並讓它們像單一 Server 一樣協同運作。

課程會依序介紹兩種 Orchestration 工具：

#### 1. Docker Swarm — 入門首選

- **優勢一**：Docker 內建，開箱即用
- **優勢二**：概念相對簡單，容易理解
- **學習重點**：YAML 檔案撰寫、多 Container 解決方案、分散式系統、Microservice 架構

#### 2. Kubernetes — 進階標準

- **為什麼先學 Swarm 再學 Kubernetes？**
  
  Kubernetes 的複雜度遠高於 Swarm。老師認為 Swarm 是很好的跳板，讓學員先建立 Orchestration 的基礎概念，再進入 Kubernetes 的學習曲線會平緩許多。

- **為什麼要學兩種 Orchestrator？**
  
  目前市面上至少有 4~6 種主流的 Orchestrator。雖然 Kubernetes 佔據了大部分的市場份額與關注度，但它並非唯一解方，有時甚至過於複雜。**擁有多種工具的知識儲備，能讓你根據實際需求做出更適合的技術選擇。**
