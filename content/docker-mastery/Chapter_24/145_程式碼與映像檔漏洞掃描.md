# 程式碼與映像檔漏洞掃描實務

## 📝 課程概述

本單元介紹如何在開發流程的早期階段導入安全掃描機制,從程式碼倉庫的依賴性檢查到容器映像檔的 CVE 漏洞掃描。我們將學習 Shift Left Security 的核心理念,以及如何選擇與實作掃描工具。

## 核心觀念與實作解析

### Shift Left Security:安全左移理念

#### 核心概念

想像一個看板(Kanban Board):
- **左側**:開發工作起始點
- **右側**:部署到生產環境

**Shift Left Security** 強調將安全檢查儘可能提早到開發流程的早期階段,而非等到部署前才檢查。

#### 實踐方式

- **最早階段**:在 Git Repository 中掃描程式碼依賴
- **中間階段**:在 CI 流程中掃描容器映像檔
- **較晚階段**:在生產環境中監控運行時行為

**關鍵效益**:愈早發現問題,修復成本愈低。

### 程式碼倉庫掃描

#### Snyk 工具介紹

**Snyk** 是知名的程式碼依賴掃描工具:

- **免費方案**:支援開源專案
- **整合平台**:GitHub、Bitbucket 等
- **掃描內容**:npm、pip 等套件管理器的依賴漏洞
- **主動通知**:每週發送漏洞報告

#### GitHub 內建掃描

現今 GitHub 已內建類似功能:
- **Dependency Graph**:自動分析依賴關係
- **Security Alerts**:發現漏洞時主動通知
- **免費使用**:公開與私人倉庫皆可使用

#### 實施建議

```yaml
# 建議同時使用多種掃描工具
- GitHub Dependabot
- Snyk
- 其他 CI 內建的掃描工具
```

### 容器映像檔掃描

#### 與程式碼掃描的差異

映像檔掃描涵蓋更廣泛的層面:

**程式碼掃描**:
- npm packages
- pip packages
- Maven dependencies

**映像檔掃描**:
- OS 套件(YUM、APK、APT)
- 程式語言依賴
- 系統函式庫
- 其他二進位檔案

#### CVE 資料庫

**CVE**(Common Vulnerabilities and Exposures):
- 由 MITRE 公司維護的社群資料庫
- 每個已知漏洞都有唯一編號
- 公開可查詢的標準化資料庫

**掃描器運作邏輯**:將映像檔中的套件版本與 CVE 資料庫比對。

#### 主流掃描工具比較

##### Trivy(推薦)

由 Aqua Security 開發:
- **最完整的支援**:
  - OS 套件:Alpine、RHEL、CentOS、Debian、Ubuntu 等
  - 語言依賴:npm、pip、Maven、Gem 等
- **解決 Alpine 問題**:許多掃描器無法正確掃描 Alpine 映像檔,Trivy 使用新方法解決此問題
- **易用性**:單一執行檔,無需安裝
- **開源免費**:完全開源且免費使用

##### MicroScanner

同樣來自 Aqua Security:
- **整合方式**:可在 Dockerfile 中加入掃描步驟
- **使用時機**:適合在建置時期掃描
- **限制**:支援範圍不如 Trivy 完整

##### Anchore

另一個流行的開源選項:
- **企業級功能**:提供更細緻的政策設定
- **部署彈性**:可自建掃描服務

#### 掃描時機的選擇

##### 1. 建置時掃描(Build Time)

```dockerfile
# 在 Dockerfile 中整合 MicroScanner
RUN microscanner [token]
```

**優點**:
- 最早期的檢查
- 確保上傳到 Registry 前已掃描

**缺點**:
- 增加 Dockerfile 複雜度
- 每次建置都需執行

##### 2. CI Pipeline 中掃描

```bash
# 使用 Trivy 掃描已建置的映像檔
trivy image myapp:latest
```

**優點**:
- 不修改 Dockerfile
- 彈性較高

**推薦做法**:在 CI 中掃描,失敗時阻擋映像檔上傳。

##### 3. Registry 掃描

- **Docker Enterprise DTR**:內建掃描功能
- **Quay**:雲端託管的 Registry,提供掃描
- **優點**:集中管理,確保所有映像檔都經過掃描

#### 重要認知:掃描結果的解讀

##### 沒有零漏洞的映像檔

**現實情況**:
- 大多數映像檔都有漏洞
- 這是開源生態的本質
- 並非所有漏洞都適用於你的應用

##### 實施建議

1. **初期採用警告模式**:
```bash
# 不要因發現漏洞就失敗,先記錄
trivy image --exit-code 0 myapp:latest
```

2. **與安全團隊合作**:
- 確認哪些漏洞必須修復
- 評估漏洞實際影響
- 制定漏洞處理政策

3. **逐步收緊標準**:
- 初期:僅記錄,不阻擋
- 中期:阻擋高嚴重性漏洞
- 成熟期:依風險評估決策

### Runtime 行為監控

#### Sysdig Falco

**Falco** 是專為容器設計的運行時監控工具:

##### 監控項目

- 執行中的容器被 `exec` 進入
- 檔案被複製到運行中的容器
- 敏感目錄被 bind mount 到容器
- 異常的系統呼叫

##### 運作方式

- 在每台伺服器上運行 Falco Agent
- 監控核心系統呼叫
- 根據規則產生警報

##### 內建規則集

Falco 提供預設的「可疑行為」清單:
- 常見的惡意行為模式
- 可自訂擴充規則
- 支援容器特定的威脅偵測

##### 部署建議

即使不立即監控,也建議部署:
- **日誌記錄**:保留行為日誌供日後追蹤
- **事件調查**:發生問題時可回溯
- **稽核需求**:滿足合規性要求

## 💡 重點摘要

- Shift Left Security 強調在開發早期階段導入安全檢查,降低修復成本
- Trivy 是目前最完整的開源映像檔掃描工具,支援 Alpine 等多種發行版
- 掃描工具初期應設為警告模式,避免因漏洞過多而阻斷開發流程
- Falco 可監控容器運行時的可疑行為,即使不主動監控也應部署以保留日誌
