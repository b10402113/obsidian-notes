# Docker Engine 內建安全功能解析

## 📝 課程概述

本單元深入探討 Docker Engine 開箱即用的安全機制,說明為何使用容器本身就是一項安全措施。我們將理解 Docker 如何預設啟用多項 Linux 安全特性,並學習使用 Docker Bench 工具來檢測宿主機的安全配置是否正確。

## 核心觀念與實作解析

### 為什麼使用容器就是安全措施?

#### 常見的安全疑慮

許多組織對於導入 Docker 存在安全疑慮,主要來自於:

- **惡意映像檔問題**:駭客在 Docker Hub 上散布含有加密貨幣挖礦程式的惡意映像檔
- **安全團隊的反對**:管理層閱讀相關報導後,對容器化產生疑慮
- **誤解本質**:將使用者不當操作歸咎於 Docker 本身

#### 核心論點:容器提升預設安全性

**使用 Docker 本身就能提升安全性**,原因如下:

1. **隔離視野**:容器內的應用程式無法看到宿主機上運行的其他程序
2. **自動啟用安全功能**:Docker 預設啟用多項 Linux 安全特性,這些在傳統部署中通常未啟用

### Docker 預設啟用的四大安全特性

Docker Engine 自動啟用以下安全機制(若宿主機支援):

#### 1. Seccomp
- 全名為 Secure Computing Mode
- 限制程序可使用的系統呼叫
- 預設封鎖危險的系統呼叫

#### 2. AppArmor
- 基於路徑的存取控制
- 限制程序的檔案存取權限
- Ubuntu 等 Linux 發行版預設啟用

#### 3. SELinux
- Security-Enhanced Linux
- 基於標籤的強制存取控制
- Red Hat/CentOS 等系統預設支援

#### 4. Linux Capabilities
- 細緻的權限控制機制
- 將傳統 root 權限拆分為數十種獨立能力
- Docker 移除不必要的危險能力

**重要觀念**:Docker 團隊研究了大量開源軟體,分析其共同需要的最小權限集合,然後移除了其他數百項不必要的權限,形成預設的安全設定檔。

### 實際案例:Nginx 的安全隔離

若直接在宿主機上運行 Nginx:
- 以 root 身份運行,可存取核心所有功能
- 擁有該使用者可存取的所有資源

若在容器中運行 Nginx:
- 自動套用安全設定檔
- 大幅限制可存取的核心功能
- **即使未自訂設定,也比傳統部署更安全**

> 這就是為什麼有時需要使用 `--privileged` 模式:因為預設的安全設定過於嚴格,與特定應用程式的需求衝突。

### Docker Bench:配置檢測工具

#### 工具定位

**Docker Bench** 是 Docker 官方提供的免費開源工具,用於檢測宿主機的 Docker 配置是否安全。

**它不是**:
- 病毒掃描工具
- 漏洞掃描工具

**它是**:
- 配置檢查工具
- 基於 CIS Docker Benchmark 標準

#### 使用時機

1. **伺服器建置階段**:在使用 Ansible、Puppet 等工具建置伺服器後
2. **測試環境驗證**:在部署到正式環境前的測試環境中
3. **定期安全稽核**:檢查現有系統的配置狀態

#### 執行方式

工具以 Shell Script 形式提供,可直接從官網執行:
- 會產生藍、綠、黃、紅等不同等級的報告
- 不可能全部都是綠燈(這是正常的)
- 管理者需逐一評估是否可接受

#### 報告解讀重點

檢測項目包括:
- Docker 是否正確安裝
- 是否具備必要的權限
- 核心能力是否完整
- SELinux/AppArmor 是否啟用
- 各項安全功能是否正確配置

**關鍵心態**:這不是為了「全部通過」,而是為了「了解現狀並做出明智決策」。

### 最佳實踐建議

1. **不要下載來路不明的映像檔**:如同不會隨意執行網路上的程式碼
2. **優先使用官方映像檔**:由廠商背書的可信賴來源
3. **信任建立於廠商信譽**:選擇信譽良好的供應商
4. **Docker Hub 僅掃描官方映像檔**:並非所有映像檔都經過掃描

## 💡 重點摘要

- Docker 預設啟用 Seccomp、AppArmor、SELinux、Linux Capabilities 四大安全特性
- 容器的預設安全設定經過大量開源軟體測試驗證,比傳統部署更安全
- Docker Bench 是配置檢查工具,可協助發現宿主機的安全設定問題
- 安全性的關鍵在於使用可信賴的映像檔來源,而非盲目禁止容器化
