## 核心觀念與實作解析

### Docker 安全基礎架構

在深入具體的安全措施之前,我們必須先理解 Docker 如何透過 Linux 核心特性來實現容器隔離:

#### 1. Kernel Namespaces

**Namespaces** 是 Linux 核心提供的基礎隔離機制,它讓每個容器擁有獨立的資源視野:

- **網路隔離**:每個容器獲得專屬的虛擬 NIC
- **檔案系統隔離**:容器無法看到宿主機的完整檔案系統
- **程序隔離**:容器內的程序無法查看或干擾外部程序
- **核心概念**:Namespaces 限制程序的「視野」與「存取權限」

> 這項技術並非 Docker 獨創,在 Unix 系統中已存在超過十年,例如 Solaris Jails 就是類似的技術。

#### 2. Control Groups (Cgroups)

**Cgroups** 主要用於資源限制,確保單一容器不會耗盡系統資源:

- **記憶體限制**:可設定容器僅使用 1GB RAM
- **CPU 限制**:限制容器可使用的 CPU 核心數
- **預設行為**:容器預設可使用所有系統資源
- **生產環境必要性**:在生產環境中必須明確設定資源限制

**為何這很重要?** 若未設定限制,單一應用程式可能消耗所有資源,導致其他服務無法運行,這類似於 DoS 攻擊的效果。
