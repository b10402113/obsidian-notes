# 多服務應用部署實作詳解

## 📝 課程概述

本單元是上一個練習的完整實作演示。我們將一步步建立 Example Voting App 的所有服務，並學習如何排查部署過程中可能遇到的問題，特別是服務啟動順序與依賴關係的處理。

## 核心觀念與實作解析

### 部署前的準備

確認 Swarm 環境：

```bash
docker node ls
```

確保有多個 Node 可用，且沒有其他運行中的服務：

```bash
docker service ls
```

---

### 步驟一：建立網路

```bash
docker network create --driver overlay backend
docker network create --driver overlay frontend
```

> 兩個網路用於隔離前端與後端服務，這是安全最佳實踐。

---

### 步驟二：建立服務

**Vote 服務**：

```bash
docker service create --name vote --network frontend --replicas 2 -p 80:80 dockersamples/examplevotingapp_vote
```

**Redis 服務**：

```bash
docker service create --name redis --network frontend redis:3.2
```

**Worker 服務**（連接兩個網路）：

```bash
docker service create --name worker --network frontend --network backend dockersamples/examplevotingapp_worker
```

**PostgreSQL 服務**：

```bash
docker service create --name db --network backend --mount type=volume,source=db-data,target=/var/lib/postgresql/data postgres:9.4
```

**Result 服務**：

```bash
docker service create --name result --network backend -p 5001:80 dockersamples/examplevotingapp_result
```

---

### `--mount` 參數詳解

`-v` 在 `docker service create` 中不支援，必須使用 `--mount`：

```bash
--mount type=volume,source=db-data,target=/var/lib/postgresql/data
```

| 參數 | 說明 |
|------|------|
| `type=volume` | 類型為 Volume（若有 `/` 則為 Bind Mount） |
| `source=db-data` | Volume 名稱 |
| `target=/var/lib/postgresql/data` | Container 內路徑 |

---

### 驗證部署狀態

**查看服務列表**：

```bash
docker service ls
```

確保 `REPLICAS` 欄位顯示正確數量（如 `1/1`、`2/2`）。

**查看 Task 分佈**：

```bash
docker service ps worker
```

---

### 常見問題：Task 啟動失敗

如果在建立服務時，依賴的服務尚未啟動，可能會看到：

```
ID         NAME       ...  ERROR
xxx        worker.1   ...  No such device or address
```

**原因**：Worker 嘗試連接不存在的 DNS 記錄（`redis` 或 `db`）。

**解決方式**：
- Docker 會自動重啟失敗的 Task
- 當所有服務都建立完成後，問題會自動解決

```bash
# 等待後再次檢查
docker service ps worker
# 應該會看到 RUNNING 狀態
```

---

### 查看服務日誌

使用實驗性功能（需在 `/etc/docker/daemon.json` 啟用）：

```bash
docker service logs worker
```

這會從多個 Node 收集日誌。

---

### 測試應用

**投票頁面**：`http://<any-node-ip>:80`

- 選擇「Cats」或「Dogs」投票

**結果頁面**：`http://<any-node-ip>:5001`

- 即時顯示投票統計（使用 WebSocket）

---

### 架構驗證

如果結果頁面能正確顯示投票數，代表：

1. ✅ Vote → Redis 連線正常
2. ✅ Worker 從 Redis 讀取並寫入 DB
3. ✅ Result 從 DB 讀取資料
4. ✅ 跨節點通訊正常

---

## 💡 重點摘要

- **服務啟動失敗時，Docker 會自動重試，等待依賴服務就緒後會自動恢復。**
- **使用 `--mount` 而非 `-v` 來設定 Service 的持久化儲存。**
- **驗證多服務應用的最佳方式是測試完整的資料流。**
- **`docker service logs` 可跨節點收集日誌（實驗性功能）。**
