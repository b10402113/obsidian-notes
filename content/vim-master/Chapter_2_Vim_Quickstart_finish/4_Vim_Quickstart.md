# Vim 入門：啟動、模式切換與基本操作

## 📝 課程概述

本單元是 Vim 學習旅程的起點，目標是讓學員在最短時間內掌握「啟動 Vim → 輸入文字 → 儲存並離開」這條核心流程，並建立對 Vim 三種基本模式的直覺認知。我們特別強調**先動手做、再看解答**的學習方式，因為「實際操作本身就是最好的學習」。

---

## 核心觀念與實作解析

### 在各系統上啟動終端機

Vim 是一款純文字編輯器，因此學習 Vim 的第一步，是先在你的作業系統上開啟一個**命令列 session（終端機）**：

| 作業系統 | 啟動方式 |
|----------|----------|
| **macOS** | 使用 Spotlight（`Cmd + Space`）搜尋「terminal」，或透過 Finder 前往 `/Applications/Utilities/` 資料夾 |
| **Windows** | 點選搜尋列，輸入「command」，點擊「Command Prompt」 |
| **Linux** | 依發行版不同，終端機可能名為 `terminal`、`console`、`GNOME Terminal`、`XTerm`、`Aterm` 或 `Rxvt`（以 CentOS 為例，可從 Applications → Utilities 找到）|

> **為什麼要先會開終端機？** 因為 Vim 是為命令列環境設計的編輯器，所有的操作都發生在這個文字介面中。熟悉終端機是使用 Vim 的前提條件。

---

### 啟動 Vim 與告別 Vim

在終端機中輸入 `vim` 並按下 Enter，即可在目前工作目錄下開啟一個全新的 Vim session。首次啟動時，你會看到 Vim 的歡迎畫面。

**首先，讓我們學會如何離開 Vim——這是每個初學者都會遇到的「卡關」問題：**

1. 確保處於 **Normal mode**（預設進入就是 Normal mode）
2. 輸入冒號 `:` 進入 **Command line mode**
3. 輸入 `q!`：其中 `q` 代表 quit，`!` 代表**強制執行**（不儲存任何變更）
4. 按下 Enter，Vim 即終止並回到終端機

```bash
:q!
```

> **為什麼需要驚嘆號 `!`？** 在 Vim 中，`:q`（quit）只在你確定沒有未儲存變更時才能執行。如果你修改了檔案卻沒有儲存，Vim 會拒絕讓你直接離開——這是一種保護機制，防止你意外丟失工作。`:q!` 的驚嘆號則是「不鳥這個保護，強行離開」。一開始你可能會常用 `:q!`，但未來當你熟悉了儲存流程後，會更常使用 `:wq` 或 `:x`。

---

### Vim 的三種模式：Normal / Insert / Command line

這是 Vim 最核心、也是最重要的概念——**Vim 不是一個「打字就能寫」的編輯器，它是一個以模式切換為核心設計哲學的編輯器。**

| 模式名稱 | 用途 | 如何進入 |
|----------|------|----------|
| **Normal mode** | 瀏覽、刪除、複製、貼上等編輯操作 | 預設狀態，或隨時按下 `Esc` |
| **Insert mode** | 實際輸入文字 | 在 Normal mode 下按下 `i` |
| **Command line mode** | 執行儲存、離開等指令 | 在 Normal mode 下輸入 `:` |

理解這個設計的**Why**非常重要：傳統編輯器只有一種模式，所有按鍵（刪除、複製、貼上、輸入）都各自佔用一個快捷鍵，導致快捷鍵眾多且雜亂。Vim 的設計思路是：**把「編輯」和「輸入」分開**——大部分時間你在 Normal mode 下用極高效的方式組織文字，需要輸入時才切到 Insert mode。

### 建立新檔案與基本編輯流程

#### 建立並編輯新檔案

```bash
vim my_data.txt
```

輸入上述指令後，Vim 會在當前目錄建立一個名為 `my_data.txt` 的空白檔案，並自動進入 **Normal mode**。

#### 開始輸入文字——進入 Insert mode

在 Normal mode 下，按下**小寫 `i`**（i = insert）即可進入 Insert mode。畫面左下角通常會顯示 `-- INSERT --` 提示，表示你現在可以正常打字了。

```text
Today I woke up and fed my three pretty cats.
```

> **為什麼是「小寫 i」？** Vim 對大小寫敏感。大寫 `I` 也是進入 Insert mode，但它的行為不同——大寫 `I` 會將游標跳到該行**行首**再進入 Insert mode，而小寫 `i` 是從**目前游標位置**進入。初學階段，建議先用小寫 `i` 即可。

如果你在打字時打錯了，直接使用 **Backspace 鍵**刪除錯誤文字並重新輸入，這與一般編輯器的操作方式相同。

#### 儲存並離開——`:wq`

完成輸入後，你需要：

1. 按下 `Esc` 返回 **Normal mode**
2. 輸入 `:` 進入 **Command line mode**
3. 輸入 `w`（write，寫入/儲存）然後 `q`（quit，離開）
4. 按下 Enter

```bash
:wq
```

`wq` 可以理解為「Write and Quit」，它會將你的變更寫入磁碟，然後關閉 Vim。

> ** `:x` 也是儲存並離開？** 是的，Vim 中 `:x` 和 `:wq` 的效果幾乎相同——都會儲存並離開。兩者的細微差異在於：如果檔案根本沒有任何變更，`:x` 不會更新檔案的 modification time，而 `:wq` 會。初學階段不用在意這個，未來自然會用到。

#### 驗證檔案是否成功儲存

儲存後回到終端機，你可以再次用 Vim 打開同一個檔案來確認內容是否正確：

```bash
vim my_data.txt
```

如果看到你剛才輸入的內容，恭喜你——你已經完成了第一個 Vim 編輯流程！

### 關於練習：先做再看

在進入實際操作練習之前，老師特別強調了**練習心態**的重要性：

> **「Doing is learning. 如果你真的動手做練習，你會保留更多的資訊。」**

建議的學習節奏：

1. **先自己想辦法做一遍**：不要急著看解答影片，給自己一個思考與摸索的機會
2. **卡住了再看影片**：看到卡關的地方之後，暫停，回到練習繼續自己完成
3. **允許彈性選擇**：你也可以先看影片再實作，或兩者同步進行——選擇最適合你的方式

## 💡 重點摘要

- **Vim 是一款以「模式切換」為核心設計的編輯器：Normal mode 用來組織文字，Insert mode 用來輸入內容，Command line mode 用來執行指令。**
- **`:q!` 強制離開（不儲存），`:wq` 儲存並離開，這兩個指令是 Vim 初學者最優先需要記住的組合。**
- **進入 Insert mode 的方式是小寫 `i`，返回 Normal mode 的方式是 `Esc`。這個 `i ↔ Esc` 的來回切換是使用 Vim 的最基本節奏。**
- **Vim 的保護機制預設不允許未儲存就離開——這是好事，它在保護你的資料不被意外丟失。**
- **學習任何工具的最佳方式都是「先動手做」，不要只是看影片——doing is learning。**
