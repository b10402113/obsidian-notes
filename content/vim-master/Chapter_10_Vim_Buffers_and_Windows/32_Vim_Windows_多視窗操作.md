# Vim Windows：多視窗操作完整攻略

## 📝 課程概述

Buffer 讓我們能在記憶體中同時管理多個檔案，而 **Window（視窗）**則是「用來看 Buffer 的視圖」。學會操作 Window，你就能**同時在畫面上查看多個 Buffer**，甚至**用多個視圖觀看同一個 Buffer 的不同部分**——這對比對大型檔案、邊看原始碼邊看文件等場景非常實用。本單元從基礎的分割開始，逐步涵蓋複雜的視窗佈局、導航捷徑、視窗大小調整，以及 `:windo` 與 `:ball` 等高效命令。

## 核心觀念與實作解析

### Window 與 Buffer 的關係

首先釐清一個常見的混淆：

- **Buffer** = 檔案的記憶體副本（一個概念）
- **Window** = 緩衝區 Buffer 的**視圖（viewport）**

一個 Buffer 可以同時出現在多個 Window 中——兩個視窗顯示同一份 Buffer，你在任一視窗做的編輯，另一個視窗會**即時同步顯示**。而 `:bn` / `:bp` 等 Buffer 命令，則是在**同一個視窗**內來回切換顯示的 Buffer。

預設啟動 Vim 時，你只有一個視窗，顯示一個 Buffer。

### 水平分割：`:split`

```
:split              " 簡寫 :sp
Ctrl+w s            " 等價操作
```

`:split` 會將當前視窗**水平切成兩半**，上下各占一半，**兩個視窗顯示同一個 Buffer**。

若想在分割時直接載入另一個檔案，把路徑傳給 split 命令：
```
:sp buf-bed.txt     " 分割並載入 buf-bed.txt
```

> 為什麼需要這個功能？例如你正在修改一個函式，需要參考同檔案另一個區塊的程式碼，就可以 `:split` 開一個相同檔案的視窗，在兩個視窗間各自滾動，互不干擾。

### 垂直分割：`:vsplit`

```
:vsplit             " 簡寫 :vs
Ctrl+w v            " 等價操作
```

與 `:split` 相同概念，但分割方向是**垂直的**——左右並排顯示。
```
:vs buf-cat.txt     " 垂直分割並載入 buf-cat.txt
```

### 在視窗間導航

Vim 提供兩套導航系統：

#### 循環切換：`Ctrl+w w`
```
Ctrl+w w    " 依序在所有視窗間循環移動
```
這個指令會依固定順序（通常是從左上到右下）在所有開啟的視窗間輪流選取，與 Buffer 的 `:bn` / `:bp` 概念類似。

#### 方向性移動：`Ctrl+w` + `hjkl`
```
Ctrl+w h    " 移到左側視窗
Ctrl+w j    " 移到下方視窗
Ctrl+w k    " 移到上方視窗
Ctrl+w l    " 移到右側視窗
```
這是 Vim 中最精確的視窗導航方式，與編輯器內的 `hjkl` 移動邏輯一致。若你經常使用多視窗，強烈建議在 `.vimrc` 中建立快捷鍵映射：
```vim
nnoremap <C-h> <C-w>h
nnoremap <C-j> <C-w>j
nnoremap <C-k> <C-w>k
nnoremap <C-l> <C-w>l
```

> **GUI 使用者的捷徑**：若使用 GVim 或 MacVim，直接用滑鼠點擊任一視窗，該視窗就會變成 active。另外，也可以用滑鼠拖曳視窗分隔線（status bar）來調整視窗大小。

### 關閉視窗

```
:quit     " 簡寫 :q，關閉當前視窗
Ctrl+w q  " 等價操作
```

**關閉除了當前視窗以外的所有其他視窗：**
```
:only     " 簡寫 :on，讓當前視窗獨占整個畫面
Ctrl+w o  " 等價操作
```

### 視窗大小調整

#### 用快捷鍵微調
```
Ctrl+w +    " 增加當前視窗高度
Ctrl+w -    " 減少當前視窗高度
Ctrl+w >    " 增加當前視窗寬度
Ctrl+w <    " 減少當前視窗寬度
```

#### 最大化與平均化
```
Ctrl+w _    " 將當前視窗高度最大化（底線 = maximize height）
Ctrl+w |    " 將當前視窗寬度最大化（豎線 = maximize width）
Ctrl+w =    " 讓所有視窗恢復為相同大小
```

### 視窗排列與移動

#### 旋轉（Rotate）
```
Ctrl+w r    " 將所有視窗向右/向下旋轉一圈（方向由分割方式決定）
Ctrl+w R    " 反向旋轉（左/上）
```

#### 移動到邊緣（移動不等於旋轉，目標位置是固定的）
```
Ctrl+w H    " 將當前視窗移到最左側（占用全部高度）
Ctrl+w J    " 將當前視窗移到最下方（占用全部寬度）
Ctrl+w K    " 將當前視窗移到最上方（占用全部寬度）
Ctrl+w L    " 將當前視窗移到最右側（占用全部高度）
```

這些指令與方向鍵 `hjkl` 的對應邏輯一致：大寫 `H` = 左（Left），`J` = 下（Down），`K` = 上（Up），`L` = 右（Right）。

### 組合水平與垂直分割：建立複雜視窗佈局

你可以將兩種分割方式混合使用，輕鬆做出「田」字形或階梯形佈局：

```
:sp buf-bed.txt       " 先水平分割，載入 buf-bed.txt
:vs buf-a.txt         " 再對新視窗垂直分割，載入 buf-a.txt
Ctrl+w w              " 切到下方視窗
:vs buf-cat.txt       " 對下方視窗再垂直分割
```

如此可以建立任意複雜度的視窗佈局，視你的工作需求自由組合。

### `:ball` — 一鍵把所有 Buffer 分配到各自視窗

若你已經開了很多 Buffer，想要一次將它們全部展示出來：

```
:ball    " 為 buffer 列表中的每個 Buffer 各開一個視窗
```

> 若螢幕空間不夠，Vim 會盡量開視窗直到塞滿螢幕，並顯示警告訊息告知有些 Buffer 無法容納。

### `:windo` — 在所有視窗中執行命令

`:windo` 與 `:bufdo` 概念相似，但作用對象是**視窗**而非 Buffer：

```
:windo %s/#/@/g
```

兩者的關鍵差異：
- **`:bufdo`**：對象是所有 **Buffer**（不管有沒有被顯示在視窗中）
- **`:windo`**：對象是所有 **Window**（每個開啟的視窗都執行一次命令）

### 關於 `Ctrl+w` 快捷鍵的兩種輸入方式

Vim 提供了兩種等價的寫法：
- `Ctrl+w Ctrl+w`：按住 Ctrl，先按 `w`，**保持按住 Ctrl** 再按第二個 `w`
- `Ctrl+w w`：按住 Ctrl，先按 `w`，**鬆開 Ctrl**，再按另一個 `w`

大多數人偏好第二種，因為不需要同時按住 Ctrl。

> **Terminal 的潛在衝突**：某些終端軟體會攔截 `Ctrl+s`（暂停螢幕輸出），導致 `Ctrl+w Ctrl+s`（相當於 `:split`）看起來完全沒反應。若遇到這種情況，按 `Ctrl+q` 恢復正常輸出即可。同樣地，`Ctrl+w Ctrl+q`（`:quit`）也可能被某些終端軟體截獲。

## 💡 重點摘要

- **視窗（Window）是 Buffer 的視圖**，同一個 Buffer 可以同時出現在多個視窗中，編輯即時同步。
- **`:sp` / `:vs`** 是最基礎也最常用的兩個分割命令，搭配檔名參數可以一步完成「分割 + 載入」。
- **`:only`** 是快速清除多餘視窗、讓工作區回歸乾淨的最佳選擇。
- **`:ball`** + **`:windo`** 是兩個隱藏寶石：前者將所有 Buffer 一次展開到螢幕，後者讓你在所有視窗中執行同一命令。
- 若常態使用多視窗，建議在 `.vimrc` 中將 `Ctrl+h/j/k/l` 映射為 `Ctrl+w h/j/k/l`，大幅提升操作效率。
