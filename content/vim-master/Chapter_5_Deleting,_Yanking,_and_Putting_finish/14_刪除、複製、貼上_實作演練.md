# Vim 刪除、複製、貼上：實作演練

## 📝 課程概述

本單元是 Chapter 5 的實作練習，目標是讓你在真實編輯場景中熟練 `dd`、`p`、`P`、`x`、`p`、`yy`、`"0` 等操作，並實際體會 Vim Register 的運作方式。課程以一個 `dpy.txt`（Delete / Yank / Put）練習檔為載體，逐步演練從簡單的行的剪切貼上，到複雜的多寄存器操作與 Undo / Redo 組合技。

---

## 核心觀念與實作解析

### 開啟練習檔

```bash
cd ~/Downloads/Vim\ class/
vim dpy.txt
```

練習檔中包含多段文字與刻意安排的「問題」，供你操作與修正。

---

### 行的剪切（Cut）與貼上（Paste）

#### 基本剪切：`dd` + `p`

```text
1. 將游標放在第一行
2. 按下 dd  → 該行被剪下，存入 unnamed register
3. 移動到目標位置（j）
4. 按下 p   → 內容從 unnamed register 貼到下一行
```

#### 同一行內容重複貼上

unnamed register 的內容**在覆蓋前可以重複使用**：

```text
dd 剪下第一行
p   貼上（游標下方）
p   再貼上一次（unnamed register 內容不變）
```

#### 行交換：`ddp`

若要將「原本第一行的內容」移到「原本第三行的上方」：

```text
dd         → 剪下（register 保有該行內容）
j j        → 移動到目標行
P（大寫）   → 貼上在**目前游標行的上方**
```

---

### 字元的交換：`x` + `p`

用 `x`（剪下單一字元）+ `p`（貼上在右方）可以交換兩個相鄰字元的位置：

```text
hte  → 游標在 h，按 x → h 被剪下，游標移至 e
       按 p → h 補上，游標位置後方 → the  ✓
```

**適用場景：**
- 修正拼字順序錯誤（常見的打字錯誤型態）
- 不需要進入 Insert mode，不用按 `Esc`，在 Normal mode 完成修正

---

### 行的複製（Yank）與貼上

#### 快速複製一行：`yy` + `p`

```text
yy   → 複製整行（存入 unnamed register）
p    → 貼上在游標下方
```

#### 複製多個 word：`2YW` + `P`

```text
I really, really love vim
       ↑ 游標在第一個 r

2YW   → 複製 "really, really "（含 word 後的空白）
P     → 大寫 P 貼上在游標**上方**
結果：I really, really, really, really love vim
```

> **`2YW` vs `2yw` 的差異**：小寫 `w` 不含 word 後的空白；大寫 `W` 以空白為邊界，複製結果更自然（貼上後兩組文字間有間隔）。

---

### Register 0 的實務應用：邊刪邊貼不受影響

#### 情境說明

當你需要連續執行「先貼上某段已複製的內容」再「刪除其他行」時，若直接 `p` 再 `dd`，unnamed register 會被 dd 的內容覆蓋——這時 Register 0 就派上用場了。

#### 操作流程

```text
"ayy    → 將 "to do" 這行存入 Register a
2dd     → 刪除連續兩行（delete this）
"0P     → 從 Register 0 貼上（仍保有最初 yank 的內容）
```

> **為什麼用 `"0` 而不用 `p`？** 因為 `p` 預設讀取 unnamed register，而 `dd` 已經把 unnamed register 覆蓋了。`"0` 存的是**最近一次 Yank 的內容**，不受 dd 影響，是「邊刪邊貼」情境下的正確選擇。

---

### 命名寄存器的追加操作

#### 存入多個片段到同一個寄存器

```text
"jyy    → 將第一行存入 Register j
"fyy    → 將另一行存入 Register f
"Jyy    → 大寫 J = 追加（Append）到 Register j
"Fyy    → 大寫 F = 追加到 Register f
```

#### 驗證追加結果

追加成功後，`"jp` 會貼出**兩行內容**（而非只有一行）：

```text
"jp   → 貼上時出現兩行，代表追加成功
```

#### 用 `:reg` 確認內容

```text
:reg j f   → 查看 Register j 和 Register f 的當前內容
```

### Undo 與 Redo 的完整行為

#### Undo：`u`

`u` 復原的是**上一個完整指令**，無論它影響了一個字元還是一百行：

```text
3dd        → 一次刪除三行
u          → 三行**全部**回來（不是只回來一行）
```

> **初學者常見誤解**：`u` 是顆粒度（granularity）為「指令」的，不是「行」的。`3dd` 是一個指令，所以 `u` 一次還原全部三行。

#### Redo：`Ctrl + r`

`Ctrl + r` 重新執行上一個被 undo 的指令：

```text
3dd        → 刪除三行
u          → 三行回來
Ctrl + r   → 三行再次被刪除（Redo）
```

#### Undo / Redo 在 Insert mode 的行為

進 Insert mode 輸入文字後，**整個 Insert session 視為一個指令**：

```text
A           → 進入 Insert mode
輸入內容...
Esc         → 返回 Normal mode
u           → **整個 Insert session 的所有內容**一次被移除
Ctrl + r    → 再次插入（Redo）
```

---

## 💡 重點摘要

- **`dd` + `p` / `P` 是最基礎的行剪切貼上；unnamed register 內容可重複使用，直到被下一個 d / y 操作覆蓋。**
- **`x` + `p` 可在不進入 Insert mode 的情況下修正相鄰字元順序錯誤，這是 Vim 文字修正的經典小技巧。**
- **`2YW`（含空白）比 `2yw` 更適合用於複製後要直接貼上的 word 區塊，結果更自然。**
- **在連續「貼上 + 刪除」的混合操作中，`"0`（Register 0）是保護最近一次 Yank 內容不被 dd 覆蓋的關鍵。**
- **`u` 復原單位是「完整指令」而非逐行；Insert mode 中整個 insert session 也是一次指令——記住這個顆粒度，能減少「undo 太多」的困擾。**
