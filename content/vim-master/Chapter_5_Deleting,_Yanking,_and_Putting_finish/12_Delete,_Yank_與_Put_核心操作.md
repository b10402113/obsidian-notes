# Vim 刪除、複製、貼上：Delete / Yank / Put 核心操作

## 📝 課程概述

本單元是 Vim 剪下、複製、貼上操作的基礎觀念建立。我們將揭開一個多數初學者不知道的事實：**Vim 的 `d`（Delete）並不是真正的刪除，而是「剪下」**。理解這件事，你才能真正掌握 Vim 的操作邏輯。此外，本單元也介紹了 `y`（Yank，複製）、`p` / `P`（Put，貼上）以及 `u` / `Ctrl+R`（復原/重做）這三組核心指令，為後續的 Registers 進階應用打下基礎。

---

## 核心觀念與實作解析

### 關鍵觀念：Delete 其實是 Cut

在多數編輯器中，「刪除」代表文字永遠消失。但在 Vim 中，`d`（Delete）以及 `x` 這兩個你已經熟悉的指令，**同時具備「刪除」與「寫入暫存區」兩個動作**——這正是 Vim 所謂的「剪下（cut）」。

這個暫存區在 Vim 裡稱為 **Register**，在其他編輯器裡則常被稱為「剪貼簿（clipboard）」。

> **為什麼 Vim 要這樣設計？** 因為 Vim 鼓勵你重複利用被刪除的文字——既然文字已經從原處移除，與其讓它消失，不如保留在 Register 裡供你隨時貼上。這是 Vim 「文字即資料」的哲學體現。

---

### 行的剪下與貼上：`dd` + `p`

#### 剪下一行：`dd`

將游標移到目標行，輸入 `dd`，該行文字會被剪下並存入 **unnamed register（未命名寄存器）**——這是 Vim 的預設 Register。

```text
原本內容：
1. 第一行
2. 第二行  ← 游標在這裡，按下 dd
3. 第三行

執行後：
1. 第三行    ← 第二行已被剪下，游標留在原位置
```

#### 貼上：`p` 與 `P`

| 指令 | 行為 |
|------|------|
| `p`（小寫） | 將 Register 內容**貼在游標所在行的下方** |
| `P`（大寫） | 將 Register 內容**貼在游標所在行的上方** |

#### 交換相鄰兩行：`ddp`

這個三連擊是 Vim 中最經典的「Swap（交換）」手法：

1. `dd` — 剪下目前這行
2. 移動到下一行（`j`）
3. `p` — 貼上

```text
執行 ddp 前：
apple
banana
cherry

執行 ddp 後：
banana
apple
cherry
```

> **為什麼 `ddp` 是值得記住的組合？** 因為 Vim 的編輯邏輯往往是「先操作文字，再移動，再操作」。`ddp` 完美體現了這個節奏——剪下 → 移動 → 貼上，三個步驟完成一行交換。

---

### 字元的交換：`x` + `p`

如果只想交換**兩個相鄰字元**的位置，例如把 `hte` 修正為 `the`，不需要進 Insert mode 重新打字，只需要：

1. 將游標放在**第一個要交換的字元**上（例如 `h` 的位置）
2. `x` — 剪下這個字元（存入 unnamed register）
3. `p` — 將其貼到游標**右側**

```text
游標在 h：
hte → 按 x（h 被剪下）→ te，游標在 e
→ 按 p（h 被貼上）→ the  ✓
```

這個技巧在處理打字錯誤時特別實用，因為多數拼字錯誤往往只是字母順序顛倒。

---

### 複製（Yank）而不刪除：`y`

`y` 是 Vim 中代表「複製」的 operator，中文稱為 **Yank**。

它的語法結構與 `d` 完全相同，都是 **operator + motion（運算子 + 動作）** 的組合：

| 指令 | 意義 |
|------|------|
| `yw` | Yank（複製）一個 word |
| `y$` | Yank 到行尾 |
| `yy` | Yank 整行 |
| `2yw` | Yank 兩個 word |
| `4yy` | Yank 四行（從游標所在行往下數） |

#### 複製整行：`yy` + `p`

將游標放在目標行，`yy` 複製整行到 register，再 `p` 貼上——等於「快速複製一行」：

```text
原本內容：
apple
banana  ← 游標在這裡，yy 複製
cherry

p 貼上後：
apple
banana
banana  ← 複製成功
cherry
```

#### 複製多個 word：`2YW` / `Y2W`

| 指令 | 行為 |
|------|------|
| `2yw` | 往下複製 2 個 word（包含標點） |
| `2YW` | 往下複製 2 個 word（**含 word 後的空白**）|
| `Y2W` | 等價於 `2YW`，只是數字放後面 |

```text
double dupe
        ↑ 游標在 double，Y2W 複製 "double dupe"
→ P（大寫）貼上：double dupe double dupe
```

> **`YW` 與 `YW` 的差異在於空白**：如果你要複製「多個 word」並直接貼上使用，**`YW`（含空白）通常更符合預期**，因為貼上後兩組文字之間會有自然間隔。

### Undo（復原）與 Redo（重做）

只要是編輯操作，就難免會出錯。Vim 提供兩個直覺的捷徑：

| 指令 | 行為 |
|------|------|
| `u` | Undo，復原上一個指令 |
| `Ctrl + r` | Redo，重做被復原的指令 |

**`u` 的復原範圍是「整個上一個指令」，而非一次一行。** 例如執行 `2dd` 一次刪除兩行，點一次 `u` 就會把**兩行一起還原**——不是逐行還原，這是許多人會誤解的地方。

```text
執行 2dd 刪除兩行 → 按一次 u → 兩行同時回來（不是只回來一行）
```

> **為什麼 `u` 一次 undo 整個指令？** 因為 Vim 的設計哲學是「每次操作都是有意義的 intent（意圖）」。你下了 `2dd` 代表你「打算刪除兩行」，所以 undo 也應該是一次undo 這個完整意圖，而不是還原成 `dd` + `dd` 的顆粒度。

---

## 💡 重點摘要

- **Vim 的 `d`（Delete）和 `x` 指令其實是「剪下（cut）」——文字被刪除的同時，也被存進了 Register，供後續貼上使用。**
- **`ddp` 是 Vim 中最經典的「交換相鄰行」組合：dd 剪下 → j 移動 → p 貼上，三鍵完成一行置換。**
- **`x` + `p` 可以用來交換兩個相鄰字元的位置，這是修正打字錯誤時最快的方法，不需要進入 Insert mode 重新輸入。**
- **`yy`（或 `y$`、`yw`）用來複製（Yank）文字到 Register；`p` / `P` 分別在游標下方與上方貼上（Put）。`4yy` 可以一次複製多行。**
- **`u` 復原的是「整個上一個指令」而非逐行還原；`Ctrl+r` 則是 Redo。**
