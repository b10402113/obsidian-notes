# Vim Registers 進階操作：Nummered / Named / Black Hole

## 📝 課程概述

本單元深入探討 Vim Register 系統的全貌。上一課我們只用到 unnamed register（預設寄存器），但 Vim 其實提供了**十個編號寄存器（0-9）**、**26 個命名寄存器（A-Z）**，以及一個特殊的**黑洞寄存器（Black Hole Register, `"_`）**。理解這些寄存器的行為差異，能讓你在複製、刪除、貼上之間自由切換，不再被「貼上錯內容」的困擾所苦。

---

## 核心觀念與實作解析

### Register 的前綴：`"`

在 Vim 中，所有 Register 的指定都以**雙引號 `"`** 開頭。例如 `""` 代表 unnamed register，`"0` 代表第 0 號寄存器，`"a` 代表命名寄存器 a。

```text
""   → Unnamed register（預設）
"0   → Register 0
"1   → Register 1
"_   → Black hole register
"a   → Named register a
"A   → 追加到 named register a
```

---

### unnamed register（`""`）：預設的剪貼簿

unnamed register 是 Vim 操作時的**預設目標**。當你不指定任何寄存器時，所有 `d`、`x`、`c`、`y` 操作的文字都會自動寫入這裡。這也是 `p` 命令**預設讀取**的寄存器。

**大多數時候你只需要用到 unnamed register 就能運作。**

---

### 編號寄存器 0-9：自動管理的歷史

Vim 維護了一組**十個編號寄存器**，用來記錄你最近的刪除與複製操作履歴，減輕「誤覆蓋」的痛點。

#### Register 0：`"0` — 最近一次 Yank（複製）的內容

當你執行 `yy` 複製一行時，該內容會同時寫入：
- **`""`（unnamed register）**： Vim 預設行為
- **`"0`（Register 0）**：專門儲存**最近一次 Yank 的內容**，不受後續 `dd`、`x` 等刪除操作影響

**為什麼這很有用？** 想像你剛 `yy` 複製了一行，隨後不小心 `dd` 刪了別的東西——此時 `""` 被覆蓋成刪除的內容，你無法直接貼上原本複製的東西。但 `"0` 仍然保留著最初 Yank 的內容，你只需要：

```text
"0P   → 從 Register 0 貼上（大寫 P 貼在游標上方）
```

#### Register 1-9：最近刪除操作的歴史棧

| 寄存器 | 內容 |
|--------|------|
| `"1` | 最近一次 Delete / Change 的內容 |
| `"2` | 上上次 Delete / Change 的內容 |
| ... | ... |
| `"9` | 第九次 Delete / Change 的內容 |

每次新的刪除或改變操作發生時，Vim 會自動將 `"1` 的舊內容推到 `"2`，`"2` 推到 `"3`，以此類推——**最早進來的內容（`"9`）會被擠出而消失**。這是一個 **LIFO（後進先出）堆疊**。

```text
dd 第一行  → "1 = 第一行
dd 第二行  → "1 = 第二行, "2 = 第一行
dd 第三行  → "1 = 第三行, "2 = 第二行, "3 = 第一行
```

**實際應用場景：** 假設你誤刪了三行東西，只要它們還在 Register 9 以內，你都可以從 `"1`、`"2`、`"3` 把它們找回來。活用編號寄存器，有時比 `u` 還要靈活。

---

### 黑孔寄存器（Black Hole Register）：`"_`

如果你想**刪除文字但不影響任何寄存器**，使用黑孔寄存器：

```text
"_dd   → 刪除目前這行（什麼寄存器都不寫入）
```

**使用時機：**
- 你 Yank 了某段文字，想要**保護它不被後續的刪除操作覆蓋**
- 你只是想單純刪除，不需要保留（這是 Vim 的慣例用法）

> **設計理念：** Vim 的哲學是「永遠提供多條路徑抵達同一個結果」。你不需要黑孔寄存器也能刪除文字，但有它讓你的意圖更精確。

---

### 命名寄存器 A-Z：完全由你掌控

除了自動管理的編號寄存器，Vim 還提供 **26 個命名寄存器（A-Z）**，你可以自由決定放什麼、什麼時候放。

#### 基本用法：指定寄存器讀寫

```text
"ayy   → 將目前這行複製到 Register a（覆蓋原有內容）
"byw   → 將一個 word 複製到 Register b
"ap    → 從 Register a 貼上（小寫 a = 讀取）
"Ayy   → 將目前這行**追加**到 Register a（大寫 A = 追加）
```

#### 大小寫的關鍵差異

| 指令 | 行為 |
|------|------|
| `"ayy` | 將該行**寫入**（覆蓋）Register a |
| `"Ayy` | 將該行**追加**到 Register a 的現有內容後方 |

```text
現有 Register a："Hello"
執行 "Ayy（追加另一行）後："Hello\nWorld"
執行 "ayy（覆蓋寫入）後："Goodbye"
```

#### 實務應用

假設你在整理一段文字，想要把**四個不同來源的片段**組合成一段新文字：

```text
1. "ayy      → 把第一個重要句子存入 a
2. "byy      → 把第二個重要句子存入 b
3. "cyy      → 把第三個重要句子存入 c
4. "ap       → 貼上 a
   "bp       → 貼上 b
   "cp       → 貼上 c
```

---

### 檢視寄存器：`:reg`

輸入 `:reg` 可以**檢視所有寄存器**的當前內容；也可以指定只看某一個寄存器：

```text
:reg        → 查看所有寄存器
:reg a      → 只看 Register a
:reg 1 2 3  → 同時看 Register 1、2、3
```

在練習刪除與複製時，`:reg` 是確認「文字到底存到哪個寄存器」的最佳工具。

---

### 與計數（Count）搭配

所有寄存器操作都可以搭配 Vim 的 count 前綴語法：

```text
2"hp   → 從 Register h 貼上兩次（等於執行兩次 "hp）
"a2p   → 從 Register a 讀取內容貼上兩次
```

---

## 💡 重點摘要

- **Register 0 (`"0`) 專門儲存最近一次 `y`（Yank）的內容；Register 1 (`"1`) 儲存最近一次 `d`（Delete）的內容——兩者不會互相覆蓋，讓你能同時保留「最後複製的東西」與「最後刪除的東西」。**
- **編號寄存器 1-9 是一個 LIFO 堆疊，每次新的刪除都會讓前一筆內容往下移一格；最先刪除的內容（`"9`）最終會被丟掉。**
- **黑洞寄存器 (`"_`) 是「環保刪除」——只刪除，不影響任何寄存器，適合想保護已 Yank 內容時使用。**
- **命名寄存器 A-Z 讓你完全掌控內容儲存位置；大寫字母（`"A`、`"B`）是追加，小寫（`"a`、`"b`）是覆蓋寫入。**
- **`:reg` 是調試寄存器內容的最佳工具，隨時可以用它確認「我想要的文字到底在哪個寄存器」。**
