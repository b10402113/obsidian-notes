# Getting Help — Vim 內建幫助系統

## 📝 課程概述

本節介紹 Vim 最核心的自學工具：`:help` 幫助系統。即使你已經記住了課程中教過的所有命令，遲早也會遇到需要查閱文件的時刻。學會使用內建幫助系統，能讓你在編輯時隨時查閱文件而不中断工作流程，真正做到**脫離課程後仍能自主學習**。

## 核心觀念與實作解析

### 開啟與離開 Help

在 Vim 中輸入 **`:help`**（或簡寫為 **`:h`**），畫面就會被分割成兩個 Window：上方是說明文件，下方是你原本正在編輯的檔案。

> **為什麼要分割視窗而不是直接覆蓋？** 因為 Vim 的設計哲學是**不中断你的工作流程**。你可以在參考文件的同時，繼續在下方視窗處理你的實際檔案。

Vim 的說明文件本質上就是純文字檔案（例如 `help.txt`），你可以像操作普通檔案一樣用 `j`/`k` 上下移動、`w` 跳躍單字。離開 Help 也很直覺，輸入 **`:q`** 即可關閉說明視窗，回到你的檔案。

### 查詢特定命令的說明

幫助系統最實用的用法，是直接查詢某個命令的具體行為。例如輸入 **`:help d`** 就會顯示 `dd` 命令的說明：

```
delete count lines into register  (ex mode)  linewise
```

這句話告訴你三件事：
- `dd` 的作用是「刪除直行（lines）」
- 模式是 **linewise**（作用於整行，而非某幾個字元）
- `count`（數量）是**可選參數**——預設刪除 1 行，加上數字前綴可一次刪除多行，例如 `3dd` 刪除 3 行

**這就是 Vim 文件中 `[brackets]` 的意義：方括號內的內容都是可選參數。** 這個規則適用於所有 Vim 說明文件。

### 查詢概念性主題

並非所有幫助項目都是命令——有時候你想了解一個**概念**，例如 `count` 或 `linewise`。這時直接輸入 **`:help count`** 或 **`:help linewise`** 即可。

`linewise` 的說明非常直白：

- 若一個命令作用於**整行**，稱為 **linewise**
- 若作用於**單一或多個字元**，稱為 **character wise**

理解這個區分非常重要，因為它影響命令的實際行為——這也是為什麼講師特別強調這是一個 **"important distinction"**。

### Help 內部的導航技巧

在文件中跳來跳去，是使用 Help 的核心技能：

| 按鍵 | 作用 |
|------|------|
| `Ctrl+O` | 跳回上一個位置（類似瀏覽器的上一頁） |
| `Ctrl+I` | 跳到下一個位置（`I` 鍵在 `O` 旁邊，方便記憶） |

另一個非常順手的技巧：將游標放在某個**關鍵字**上，直接按 **`` Ctrl+] ``**（Control + 右方括號），就會立刻跳到該關鍵字的說明。例如把游標移到 `linewise` 上按 `` Ctrl+] ``，就會直接跳到 `linewise` 的說明章節。

### 視窗切換：同時閱讀文件與參考幫助

當 Help 視窗開啟時，游標預設在上方的 Help 區。如果你想切回下方的編輯檔案，按 **`` Ctrl+w Ctrl+w ``**（保持 Ctrl，按 `w` 兩次）。此後重複相同指令，就能在兩個視窗之間來回切換。

> **為什麼要會這招？** 因為你可以在看說明的同時，直接在下方視窗演練文件中介紹的命令。邊看邊做，學習效率最高。

### Tab 補全與 Command Line Completion

當你只記得命令的開頭，想不起完整名稱時，Vim 提供了兩個好用的補全工具：

#### `Ctrl+D` — 列出所有匹配選項

```
:h :q<Tab>
```

輸入 `:h :q` 後按 `Ctrl+D`，畫面會列出所有以 `:q` 開頭的可用命令（如 `:q`、`:qa`、`:quit`）。這讓你在忘記完整名稱時也能找到目標。

#### `Tab` — 逐步輪巡候選

輸入 `:h :qu` 後不斷按 `Tab`，Vim 會依序自動補全：`quit` → `quitall` → `quote` → `quit`（迴圈）。

**`wildmenu` 選項**（若已開啟）會在 Command Line 上方額外顯示一個候選清單，並高亮當前選項。用 **Shift+Tab** 可以反向輪巡。

```
:set wildmenu     # 開啟
:set nowildmenu   # 關閉
```

如果你的 Vim 環境中看不到 wildmenu 的清單效果，別緊張——只是這項設定預設關閉，補全功能本身仍然正常運作。

### 理解文件中 Optional 的表示法

回到 `:help :quit` 的說明，你會注意到：

```
:[range]q[uit][!]
```

其中 `[range]`、`[uit]`、`[!]` 都是方括號包住的字元，代表**這些字元都可以省略**：
- `q` 是 `quit` 的簡寫
- `:` 是 `:help` 語法的一部分（也可以省略）

所以 `:q`、`:quit`、`:q!` 都是完全等价的指令。

## 💡 重點摘要

- **`:help`**（或 **`:h`**）開啟幫助系統，`:q` 關閉幫助視窗
- **方括號 `[...]` = 可選參數**，`count` 這類出現在命令說明中的關鍵字本身也可以單獨查 `:help count`
- **`` Ctrl+] ``** 跳到游標所在關鍵字的說明，**`Ctrl+O`/`Ctrl+I`** 前進／後退歷史
- **`Ctrl+w Ctrl+w`** 在 Help 視窗與編輯視窗之間來回切換
- **`Tab` 補全 + `Ctrl+D` 列出選項**，是快速查詢命令的最佳組合

## 🔑 關鍵字

`:help`, `:h`, `Ctrl+O`, `Ctrl+]`, `Ctrl+D`, Tab completion, `wildmenu`, linewise, character wise, count, window split
