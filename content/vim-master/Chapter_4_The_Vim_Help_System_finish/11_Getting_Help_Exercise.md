# Getting Help Exercise — 實作練習與常用命令總覽

## 📝 課程概述

本節是 Chapter 4 的實作練習課。我們打開 Vim 課程附帶的 `help.txt` 檔案，實際操作 Vim 的內建幫助系統，透過查詢已學過的命令（如 `i`、`wq`、`q`、`Ctrl+F`）來熟悉 `:help` 的各種用法。這個練習的重點不是背命令，而是建立**查文件的肌肉記憶**。

## 核心觀念與實作解析

### 練習環境設定

練習一開始，先在終端機導航到 Vim Class 資料夾，打開附帶的 `help.txt` 檔案：

```bash
cd ~/Downloads/vim-class
vim help.txt
```

進入 Vim 後，輸入 **`:h`**（`:help` 的簡寫）進入幫助系統。之後所有的練習都在 Help 視窗內完成。

### 練習：查詢已學過的命令

我們逐一查閱幾個課程中已經接觸過的命令，驗證 `:help` 確實能提供正確的說明。

#### `i`（Insert 模式）

```vim
:h i
```

說明寫道：`i` 在游標**前方**插入文字，並進入 Insert 模式。這與 `a`（append，在游標**後方**插入）形成對稱——理解這個差異有助於你在不同情境下選擇更順手的命令。

#### `:wq`（寫入並退出）

```vim
:h :wq
```

`:wq` 的意思是**將目前的檔案寫入磁碟後關閉視窗**。從說明文件中可以看到，`w` 和 `q` 各自獨立的用法也存在——`:w` 只寫入，`:q` 只退出。把它們組合成 `:wq` 只是常見的快捷組合。

#### `:q`（退出）

```vim
:h :q
```

`:q` 關閉**當前視窗**。在 Help 練習中，`:q` 只會關閉上方的 Help 視窗，下方的編輯視窗仍然存在。如果只剩一個視窗，再執行 `:q` 才會真正離開 Vim。

> **常見混淆**：許多初學者誤以為 `:q` 一定會退出整個 Vim。其實 Vim 可以開多個視窗，`:q` 只作用於「當前」的視窗。

#### `Ctrl+F`（往前翻頁）

```vim
:h ^F
```

`Ctrl` 鍵在 `:help` 中的簡寫是 **`^`**（Shift+6），所以查 `Ctrl+F` 就是 `:h ^F`。

> **另一個敲法**：在 Vim 中，`Ctrl` 鍵經常寫作 `^`，後面接實際按鍵。例如 `Ctrl+B` 寫作 `^B`，`Ctrl+W` 寫作 `^W`。`^F` 就是向前（Forward）翻一頁，`^B` 是向後（Backward）翻一頁。

#### `w` 與 `W`（單字跳躍）

```vim
:h w
```

`:h w` 帶你到 **word-motion** 說明區。小寫 `w` 一次跳躍一個由空白和標點分隔的單字。大寫 `W` 的行為類似，但只以**空白字元**（空格、Tab）作為分隔邊界，因此 `W` 在處理含底線或連字符的識別符時，會視為單一單位跳躍。

### 視窗切換實戰

練習中反覆用到這個操作：

```
Ctrl+w Ctrl+w   # 切到下一個 window
```

一開始游標在 Help 視窗，按一次 `Ctrl+w Ctrl+w` 就跳到下方的編輯視窗，再按一次又回到 Help 視窗。在本練習中，這個組合讓你在查說明的同時，不時確認自己正在編輯 `help.txt` 這個檔案。

> **確認目前檔案**：`Ctrl+G`（或 `:f`）可以在 Command Line 顯示目前檔案的名稱。這在同時開多個檔案時特別有幫助。

### 透過 `` Ctrl+] `` 快速跳轉

在 Help 文件中四處移動時，試著把游標移到某個關鍵字上，例如 `count`，然後按 **`` Ctrl+] ``**，Vim 會直接跳到 `count` 的說明處。按 **`Ctrl+O`** 就能跳回剛才的位置。

這種「游標移到哪就跳到哪」的方式，比每次都重新輸入 `:h 關鍵字` 更加直覺。

### `Ctrl+G` 與 `:f` 的等價性

在練習中，講師用 `:f` 查詢目前檔案，畫面底部顯示 `help.txt`。然後提到 `:f` 與 `Ctrl+G` 是**完全等價**的命令——在大多數 Vim 設定中，`Ctrl+G` 只是 `:f` 的鍵盤快捷鍵。兩者的輸出內容相同：

```
"help.txt" [New File] --100%-- col 1
```

### 探索更多：不同方式執行同一命令

練習中特別指出：Vim 允許多個按鍵組合錶達同一意圖。例如：

- `:q` = `:quit`（兩者完全相同，`uit` 為可選）
- `Ctrl+G` = `:f[ile]`（`ile` 為可選）

這不是 Vim 的例外，而是一種**一致的設計語言**：你在說明文件中看到的任何 `[可選]` 部分，都代表可以省略而不影響命令的核心功能。

> **為什麼要同時教簡寫和完整寫法？** 因為了解 `[可選]` 的意義後，你不會被兩種寫法搞混，反而能選擇更適合當下情境的版本——寫腳本時用完整寫法（可讀性更好），日常操作時用簡寫（速度更快）。

## 💡 重點摘要

- **`:h ^F`**（或 `:h Ctrl+F`）查詢 `Ctrl+F` 的說明，`^` 是 `Ctrl` 在 `:help` 中的縮寫符號
- **`:q` 關閉當前視窗，不是整個 Vim**——只剩一個視窗時 `:q` 才會真正離開
- **`Ctrl+w Ctrl+w`** 來回切換視窗，`:f` 或 `Ctrl+G` 確認目前檔案名稱
- **`` Ctrl+] ``** 直接跳到游標所在關鍵字的說明，`Ctrl+O` 返回
- Vim 同一命令常有**多種等價寫法**，這是設計選擇，不是例外
