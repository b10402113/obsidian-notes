# Vim 刪除文字與 Operator-Motion 思維

## 📝 課程概述

本單元是 Vim 編輯實力的真正起點。我們從最直觀的「按 X 刪除字元」出發，迅速過渡到 Vim 最核心的設計概念——**Operator-Motion 模式（運算子 + 動作）**。一旦掌握這個框架，你就從「死記一堆獨立指令」進化到「用組合邏輯自由創造命令」，這是 Vim 與其他編輯器最本質的差異。

---

## 核心觀念與實作解析

### 兩個最基本的刪除指令：X 與 x

| 按鍵 | 刪除什麼 | 說明 |
|------|----------|------|
| `x`（小寫）| 刪除**游標下方**的那個字元 | 在游標上按下去，那個字消失 |
| `X`（大寫）| 刪除游標**左側**（之前）的那個字元 | 游標本身不動，左邊的字被砍掉 |

> **為什麼需要兩個？** 想像你在修正一個拼字錯誤：`mstake` → `mistake`（多了一個 s）。你的游標在 s 上，按 `x` 可以刪除那個多餘的 s。但如果你的游標在 s 的右邊（i.e. 在 t 上），按 `X` 才能把那個 s 從左側刪掉。這兩個指令讓你無論從哪個方向切入都能靈活修正。

**注意一個關鍵行為**：大寫 `X` **不會刪除游標本身**——它只會刪除游標左側的字元。這在你想保留某個字元不動、只想修掉它左邊內容時特別有用。如果要刪除游標下的字元，就用 `x`。

---

### Operator-Motion 模式：Vim 的「組合語言」

這是本課最重要、也是 Vim 整個設計哲學的核心概念。

#### 拆解 `dw`（delete word）

當你輸入 `dw`（按 d，再按 w）時，Vim 內部是這樣處理的：

```
d  →  operator（運算子）：這裡是「刪除」
w  →  motion（動作）：這裡是「跳到下一個 word 的開頭」
合起來 = 刪除從游標位置到下一個 word 開頭之間的所有內容
```

也就是說：`dw` = **Delete to the next word boundary**，刪除從當前位置到下一個 word 開頭的所有字元。

> **為什麼這個設計如此強大？** 因為你不需記憶「DW」這個整體指令——你只需要知道兩件事：1. `d` 是刪除運算子；2. `w` 是「移動到下一個 word」。兩者組合就是 `dw`。這個模式讓你可以用**有限的 operator** 搭配**任意 motion** 來自由創造新命令。

#### 同一模式的更多驗證

| 組合 | 意義 | 效果 |
|------|------|------|
| `dl` | delete + 向右一字元 | 等同於 `x`（刪除游標下的字元）|
| `dh` | delete + 向左一字元 | 等同於 `X`（刪除左側字元）|
| `dj` | delete + 下一行 | 刪除**當前行 + 下一行** |
| `dk` | delete + 上一行 | 刪除**當前行 + 上一行** |
| `d0` | delete + 到行首 | 刪除從游標到行首的所有內容 |
| `d$` | delete + 到行尾 | 刪除從游標到行尾的所有內容 |

> **`x` 竟然是 `dl` 的捷徑！** 這就是 Vim 的另一個特性：有時候同一個操作會有好幾種達成方式。`x` 等同於 `dl`，`X` 等同於 `dh`，`D`（大寫）等同於 `d$`。了解這些捷徑可以加速操作，但更重要的是理解背後的 Operator-Motion 邏輯——一旦你懂了原理，捷徑自然就來了。

---

### 刪除整行：DD 與 D（大寫）

`dd`（按 d 兩次）是一個特殊情況——它刪除**整行**，不受游標在該行位置的影響。`D`（大寫）是 `d$` 的捷徑：刪除從游標到行尾的所有內容。

---

### Count（計數）+ Operator = 重複執行

Vim 允許在任何指令前加上一個數字（N），讓該指令重複 N 次：

| 組合 | 意義 |
|------|------|
| `3dd` | 刪除 3 行 |
| `5dw` | 刪除 5 個 word |
| `2d3w` | `d3w` 重複執行 2 次 = 刪除 6 個 word |

> **不指定數字時，預設為 1。** `dw` = `1dw`，`dd` = `1dd`，所以 `3dd` 與 `dd` 的行為差異僅在於數字。

另外，`N` 本身也可以單獨使用（例如 `3w` = 向前跳 3 個 word），這個能力讓 count 不僅限於 operator，可以配合任意 motion 使用。

---

### 句點命令（Dot Command）：最貼心的「再做一次」

**句點命令 `.`** 會**重複上一次執行的指令**。

實用場景：刪除一段連續文字中的多個相同模式時，你不需要每次都輸入 `dd`，只需要：
1. 按一次 `dd` 刪除第一行
2. 然後不斷按 `.`，Vim 會自動重複「刪除一行」這個動作

```bash
dd    # 刪除第一行
.     # 再刪除一行（等於 dd）
.     # 再刪除一行
.     # 再刪除一行
```

> **句點命令是 Vim 中最有「黏性」的操作之一。** 它記住的是「上次執行的完整指令」，包括任何 count + operator + motion，所以如果你上次執行了 `3dw`，`.` 就會再次刪除 3 個 word。

---

### 三個驚嘆號 `!` 的用途（額外補充）

在談論 `:q!` 時，老師順帶說明了 Vim 中 `!` 的三種常見用法，本課已涵蓋兩種：

| 用法 | 範例 | 意義 |
|------|------|------|
| 強制執行 | `:q!` | 強制執行，忽略警告（例如放棄未儲存的變更）|
| 切換開關 | `:set ruler!` | 將某個設定在 on/off 之間切換 |
| 執行外部命令 | `:!ls` | 在 Vim 內執行外部 Shell 命令 |

> **`!` 的直覺意義：「忽略一切，就照我說的做。」** 這是 Vim 中 `!` 最常見的心理模型。

---

## 💡 重點摘要

- **`x` 刪除游標下的字元，`X` 刪除游標左側的字元——兩者方向相反，使用時要根據游標位置選擇。**
- **Operator-Motion 是 Vim 的核心設計哲學：operator（d）搭配 motion（w, l, j, $ 等）產生各種編輯操作，一旦理解這個組合邏輯，你幾乎可以用現有指令達成任何想要的編輯效果。**
- **`dd` 是刪除整行的快捷鍵，`D`（大寫）是刪除到行尾的快捷鍵，它們都是 Operator-Motion 的特例化快捷鍵。**
- **在指令前加 count（N）可以讓操作重複 N 次：3dd、5dw、2d3w 等；`.`（句點）則是「再做一次上一個指令」的終極懶人鍵。**
- **Vim 中同一個目標通常有多種達成方式——理解底層邏輯比死記捷徑更重要，學會原理後捷徑自然浮現。**
