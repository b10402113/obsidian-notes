# 練習 02 導航指令實作演練

## 📝 課程概述

本單元是「Vim 必學導航指令」的實作練習 walkthrough影片。老師帶領學員逐一操作 `nav.txt` 練習檔，實踐 HJKL 逐行移動、按頁跳轉、word / WORD 跳躍、以及行號跳轉等各項指令，在操作中強化前一堂課所學的肌肉記憶。

## 核心觀念與實作解析

### 開啟練習檔

與前一堂課相同，先用命令列導航到下載檔案的所在位置：

```bash
cd ~/Downloads
cd vimclass
vim nav.txt
```

---

### 逐行與逐字元移動實作

#### HJKL：游標在列內會保持欄位

使用 `j` 向下移動時，有一個值得注意的行為：**當游標移到一個空白行時，它會停在該行開頭（column 1），但如果繼續按 j，它會**嘗試維持原本的欄位位置**（column position）**。這說明 Vim 在縱向移動時，預設會保持橫向座標。

用 `h` 將游標移到行首，然後按 `k` 回到檔案頂端——反覆練習這些基礎動作是建立直覺的好方式。

#### Ctrl + F / Ctrl + B：逐頁

按住 `Ctrl + f` 可以一路翻到檔案底部；`Ctrl + b` 向上翻一頁。不同螢幕大小的同學，每次翻頁的行數會不同，這完全正常。

---

### Word 與 WORD 移動的實際差異

這裡老師以一行包含標點的文字為例，親自演示 `w`/`W` 和 `b`/`B` 的行為差異：

#### 小寫 w / b：停在標點上

假設文字為 `far, far, far away`：

- 按 `w`：far → `,` → far → `,` → far → `,` → away
- 按 `b`：away ← `,` ← far ← `,` ← far ← `,` ← far

每次都會把標點視為獨立的 word停下來。

#### 大寫 W / B：忽略標點，以空白為界

使用 `Shift + w`（即 `W`）：

- `far, far, far away` 中，會跳過逗號，直接停在 `far`、`far`、`far`、`away`
- 按 `B` 回退時，同樣直接跳回上一個單字，忽略標點

#### 實作練習中驗證的事

- 按 `w` 8 次停在逗號；按 `b` 8 次回到開頭
- 按 `W` 跳過逗號，按 `B` 跳回時也一樣

> **記憶技巧：** 小寫指令是「瑣碎的（punctuation-inclusive）」，大寫指令是「包容的（punctuation-exclusive）」。

---

### 行號跳轉實作

#### gg / G：回到頂端 / 去到底部

按 `gg`，立刻回到檔案第一行；按 `Shift + g`（即 `G`），跳到檔案最後一行。

#### ngg / nG：跳到指定行

- **`1gg`** 或 **`1G`**：跳到第一行（與 `gg` 等效，但 `1gg` 比較冗餘，通常直接用 `gg`）
- **`29gg`** 或 **`29G`**：跳到第 29 行
- **`3G`**：跳到第 3 行

> **練習建議：** `3G` 和 `3gg` 效果相同。`:3`（在 Command line mode）也可以，但 `nG` 對手指來說更順手。

#### :4 與 :$：在 Command line mode 跳行

- **`:4` + Enter**：跳到第 4 行
- **`:$` + Enter**：跳到檔案最後一行（`$` = 正規表達式中的「結尾」）

#### 驗證位置：Ctrl + G

按 `Ctrl + g` 檢查當前所在行與總行數。如果已開啟 `:set ruler`，狀態列會顯示 `44`（行號），否則會顯示完整的 `line X of Y` 訊息。

---

### 實作後的延伸練習

老師建議在離開練習檔之前，多花幾分鐘自由探索：

- 反覆用 `w` / `W` 比較同一行的跳躍行為
- 用 `gg` / `G` 來回練習，感受大寫 G 的「默認到最後一行」設計
- 用 `Ctrl + g` 和 `:set ruler` 觀察兩種位置資訊的差異

完成所有練習後，**不儲存直接離開**：

```bash
:q!
```

---

## 💡 重點摘要

- **`j` 向下移動時，會盡量保持原來的 column 位置——空白行例外，會回到 column 1。**
- **`w` / `b` 會在標點上停下來；`W` / `B` 直接以空白為界，忽略標點。這個差異在處理程式碼時特別重要。**
- **`gg` 和 `G` 分別是「到第一行 / 到最後一行」的快捷鍵，是 Vim 中最常用的導航指令之一。**
- **`1G` 和 `gg` 功能相同，往後直接用 `gg` 即可，`:1` 在 Command line mode 也是同樣效果。**
- **`:$` 可以跳到檔案最後一行，即使你不知道檔案有多少行——這在處理大檔案時非常實用。**
