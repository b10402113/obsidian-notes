# Vim 必學導航指令

## 📝 課程概述

本單元是 Vim 學習的第一個實戰里程碑——掌握在檔案內高效移動游標的能力。老師在正式進入編輯技巧之前，先花一堂課的篇幅強調導航，原因很實際：**無論你是系統管理員修改設定檔，還是開發者要修復 Bug，第一件事都是「抵達目標位置」，因此導航技能往往比進階編輯技巧更直接影響工作效率。**

---

## 核心觀念與實作解析

### 單一字元 / 單一行移動：HJKL 四鍵

在 Normal mode 下，有四個最基礎的移動指令：

| 按鍵 | 移動方向 | 記憶方式 |
|------|----------|----------|
| `h` | 向左一格 | 在鍵盤上，H 在左側 |
| `j` | 向下一行 | 長得像**向下的魚鉤**或**下箭頭** |
| `k` | 向上一行 | 唯一剩下的，自然向上 |
| `l` | 向右一格 | 在鍵盤上，L 在右側 |

按住任一鍵，游標會持續向該方向移動。

> **為什麼要學這四個鍵而不是用方向鍵？** Vim 的核心理念是「**手不離開主鍵盤區（home row）**就能完成所有操作」。學會 HJKL 後，你的雙手可以一直保持在打字位置，不必伸手去碰方向鍵——這是效率的差異。更重要的是，在某些遠端連線或特殊環境下，方向鍵可能無法運作，但 HJKL 永遠可用。

---

### 逐頁移動：Ctrl + F 與 Ctrl + B

| 按鍵組合 | 功能 | 記憶提示 |
|----------|------|----------|
| `Ctrl + f` | 向下翻頁（Page Down） | f = **f**orward，往前翻 |
| `Ctrl + b` | 向上翻頁（Page Up） | b = **b**ackward，往回翻 |

持續按住 Ctrl 再重複按 F 或 B，可以連續翻頁。搭配 `zz` 或 `z + Enter`（將目前游標行移到螢幕中央 / 頂部），可以隨時優化可視區域。

---

### 波浪號（~）：那些「不是檔案內容」的符號

當你來到檔案底部時，可能會看到一些以 **`~`**（波浪號）開頭的行：

```
...（檔案內容的最後一行）


~
~
```

這些波浪號**不是檔案內容的一部分**。它們是 Vim 的視覺提示，用來告訴你：「這裡是檔案的終點，底下沒有任何東西了。」最後一個沒有波浪號的空白行，才是真正的檔案結尾。

---

### 以「文字單位」為粒度移動：Word 與 WORD

Vim 將文字劃分為兩種單位：

- **`w`**（word）：連續的非空白字元組成一個 word，**包含標點符號**（例如 `hello,` 會視為 word + punctuation）
- **`W`**（WORD）：純粹以空白（空格、Tab）為邊界，大寫版會**忽略標點符號**

| 按鍵 | 行為 | 遇到 `hello,world` 時停在 |
|------|------|--------------------------|
| `w` | 跳到下一個 word（含標點） | 停在 `,` |
| `W` | 跳到下一個 WORD（忽略標點） | 跳過 `,` 直接到 `world` |

回退時同理：`b`（小寫）會停在標點上，`B`（大寫）則跳過標點、以空白為界。

> **大小寫對照是一個規律：** Vim 中，**小寫指令考慮標點符號作為邊界，大寫指令忽略標點、以空白為邊界**。這個規律在 Vim 裡會一再重複出現（例如 `c`、`i`、`a` 等運算子），值得現在就記住它。

---

### 行內快速跳躍：行首、行首第一個字元、行尾

| 按鍵 | 目的地 | 說明 |
|------|--------|------|
| `0`（數字零） | 行真正的最左側 | 包含該行的所有縮排（Tab 或空格）|
| `^`（Shift + 6） | 行內第一個**非空白字元** | 若熟悉正規表達式，^ 代表「行首」的意義 |
| `$`（Shift + 4） | 行的最右側（最後一個字元**之後**）| 正規表達式中 $ 代表「行尾」 |

---

### 跳到特定行：GG 與 G，以及 :N 指令

Vim 提供多種跳轉行號的方式：

| 指令 | 行為 |
|------|------|
| `gg` | 跳到檔案**第一行**（預設值）|
| `G`（大寫）| 跳到檔案**最後一行**（預設值）|
| `ngg` 或 `nG` | 跳到第 `n` 行（例如 `11G` 跳到第 11 行）|

在 Command line mode（`:）下也可以用 `:n` 直接跳到第 n 行。兩個特別有用的指令：

- **`:1`** — 跳到第一行
- **`:$`** — 跳到檔案最後一行（`$` 來自正規表達式，意為「結尾」）

---

### 查詢檔案狀態與游標位置：Ctrl + G 與 :set ruler

#### Ctrl + G

在 Normal mode 下按 `Ctrl + g`，狀態列會顯示：

```
"nav.txt" [+] [New DIRECTORY] 74L, 3472C --67%--  ,67  col 1
```

資訊包括：
- **檔案名稱**與修改狀態
- **總行數**（74L = 74 Lines）
- **目前所在行 / 總行數**
- **檔案進度百分比**（`--67%--`）
- **目前欄位（column）**

> 請注意：這個資訊是**按 Ctrl+G 當下**的快照，隨後移動游標時不會自動更新。

#### g + Ctrl + G（g Ctrl-G）

如果你需要更詳細的統計資訊（字元數、詞彙數），可以使用 `g + Ctrl + g`，會顯示更完整的計數。

#### :set ruler

在 Command line mode 輸入 **`:set ruler`**，畫面右下角會**持續顯示**游標的行列座標（例如 `67,1` = 第 67 行、第 1 欄），無需每次都按 Ctrl + G。

切換的方式：
- **`:set ruler`** — 開啟
- **`:set noruler`** — 關閉
- **`:set ruler!`** — 切換（toggle）

這個 `:set` / `:set no` 前綴是 Vim 設定選項的通用語法：**`:set`** 開啟，`**:set no`** 關閉，`!` 切換。

---

## 💡 重點摘要

- **HJKL 是 Vim 的四方向移動基礎鍵，設計初衷是讓雙手不必離開主鍵盤區，長期使用比方向鍵更高效。**
- **w / W / b / B 四個 word 移動指令，核心差異在於是否將標點符號視為 word 邊界；記住「小寫看標點，大寫看空白」這個規律。**
- **Ctrl + F / Ctrl + B 是逐頁移動，`:n` / `:$` 是行號跳轉，`:set ruler` 是隨時掌握位置的利器。**
- **Vim 中一個重複出現的模式：大小寫往往代表「是否包含標點」的兩種版本，學會這個對應關係可以一通百通。**
- **在 Normal mode 下，幾乎所有操作都不需要伸手離開鍵盤——這是 Vim 哲學的核心。**
