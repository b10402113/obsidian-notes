# 練習 03 刪除文字實作演練

## 📝 課程概述

本單元是「刪除文字與 Operator-Motion 思維」的實作 walkthrough。老師以 `practice_deleting.txt` 練習檔為載體，逐一實踐 `x`、`X`、`dw`、`dd`、count 與 dot command 等各項刪除操作，並在過程中強化「Operator-Motion」與「count + operator + motion」這兩套 Vim 組合邏輯的直覺。

---

## 核心觀念與實作解析

### 開啟練習檔

```bash
cd ~/Downloads
cd vimclass
vim practice_deleting.txt
```

---

### 修正拼字錯誤：x 指令的應用

目標：修正第三行 `mistake`（多了一個 K）。

操作邏輯：
1. 按 `$` 跳到行尾
2. 按 `h` 兩次回到多餘的 K 上
3. 按 `x` 刪除游標下的字元

> **選擇 `$` 而非一直按 L 的原因：** 老師在這裡選擇 `j` + `$` + `h` 兩步完成跳躍，而不是一路按 L 十幾次。這體現了 Vim 的效率思維：**選擇能減少重複按鍵的更大跳距**。隨著你學到的導航指令越多，這種「選擇最有效路徑」的直覺會越來越敏銳。

---

### 刪除同行多個目標：x 的連續應用

第四行的多個單字各有重複字母（例如 `lettters`、`worrds`），操作方式：

1. 按 `0` 跳到行首（數字零）
2. 按 `l` 移動到目標字元
3. 按 `x` 刪除多餘字母
4. 對每個單字重複以上步驟

---

### 刪除到行尾：X 與 D

#### 用 X 從右側往左刪

按 `$` 將游標移到行尾（`$` 停在**最後一個字元之後**），然後按住 `Shift + x`（即 `X`），會從右側往左一個一個刪除——但**最後一個字元（即游標下的那個）不會被刪掉**。

> **為什麼最後一個字元還在？** 因為 `X` 刪除的是「游標左側」的字元。游標在最右邊時，左側所有字元都會被刪除，但游標正下方的字元不在左側範圍內，所以它留存下來。

#### 用 D（Shift + d）一次性刪除到行尾

如果你的目標是「從目前位置一口氣刪到行尾」，**`D`（`Shift + d`）比反覆按 `X` 更有效率**。`D` 等同於 `d$`（Operator-Motion 語法：刪除 + 到行尾的 motion）。

---

### 刪除行內特定範圍：d + motion 的實際應用

#### 刪除游標**右側**的一個 word

按 `dw`：從游標位置刪到下一個 word 開頭。

#### 刪除包含標點與空白的精確範圍

若要刪除 `away,`（包括逗號和後面的空格），該怎麼做？

- `dw`：停在標點前（`away` 後停下，不包含逗號與空格）
- `D` + `Shift + w`（`DW`）：使用大寫 W motion（忽略標點），一口氣刪除 `away, `

```bash
DW
```

> **`DW` = Delete WORD（大寫 WORD 會忽略標點，以空白為邊界）。** 這與 `dw` 的差異正是小寫/大寫 motion 的核心應用場景。

#### 反向刪除：d + b

`db`（delete backward）從游標往回刪除一整個 word（包括標點）。

#### 刪除多個 word：d + count + motion

| 目標 | 組合鍵 | 說明 |
|------|--------|------|
| 刪除 2 個 word | `2dw` 或 `d2w` | count 在 operator 前或 motion 前皆可 |
| 刪除 2 個 WORD | `2dW` 或 `d2W` | 使用大寫 W，忽略標點 |

> **count 放在 operator 前或 motion 前，結果相同。** `3dd` = `d3d`，`5dw` = `d5w`。Vim 對此是自由的，你可以選擇最順手的語序。

---

### 刪除整行與多行：DD 與 count

#### 刪除單行：`dd`

游標在行上任何位置，按 `dd` 即刪除整行。

#### 刪除多行：`3dd`

`3dd` = 刪除當前行以及緊接在下面的 2 行，共 3 行。**包含空白行在內**——只要是在 count 範圍內的每一行，都會被刪除，不管該行是否為空白。

---

### 句點命令（.）的終極應用

如果上一個指令是 `3dd`，那麼按一次 `.`（句點）就等於再執行一次 `3dd`。

```bash
3dd    # 刪除 3 行
.      # 再刪除 3 行（等同再次執行 3dd）
```

這就是為什麼老師說「如果你想用一次按鍵刪除多行，先用 count 執行一次，然後用 `.` 重複」。一旦你建立了 `3dd` 這個操作，用 `.` 就可以無腦重複。

> **句點命令不只適用於刪除。** `.` 會重複**任何上一次執行的完整指令**——包括 motion、count、以及所有參數。所以如果你上次執行了 `d3w`，`.` 會再次刪除 3 個 word。

---

### 退出練習檔的方式

- **儲存並離開：`:wq`**
- **放棄變更並離開：`:q!`**

---

## 💡 重點摘要

- **`X` 刪除左側字元、停在最後一個字元；`D`（`d$`）從游標一口氣刪到行尾——兩個指令各有適用場景，視你從哪個方向切入而定。**
- **`DW` 使用大寫 WORD motion，可以一次性刪除包含標點的完整單位（如 `away, `），比先 `dw` 再補空格更有效率。**
- **count 可以放在 operator 前或 motion 前：`3dd` = `d3d`，`5dw` = `d5w`——Vim 對語序很寬容，選自己順手的就好。**
- **`3dd` 後按 `.` 會再次刪除 3 行；`.` 忠實重現上一次完整的指令組合，這是 Vim 最常用的高效操作之一。**
- **在刪除前，先用導航指令（如 `$`、`w`、`b`）選好「從哪裡開始刪、刪到哪裡停下」，是 Operator-Motion 思維的正確打開方式。**
