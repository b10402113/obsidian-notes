# Vim 即時 Q&A 直播第一場

## 📝 課程概述

本單元是 Vim Master Class 的第一場即時問答直播，Jason Cannon 老師回應了學員課前預先提交的幾個常見問題，涵蓋終端機多工切換、GVim 與 CLI Vim 的抉擇，以及 Vim 在程式開發以外的文字處理場景，幫助學員將 Vim 技能與實際工作流整合得更順暢。

## 核心觀念與實作解析

### 在 Vim 與終端機之間快速切換

不少學員疑惑：「我要如何在 Vim 與終端機之間快速來回切換，執行完命令再回到編輯器？」

老師提供了三種層次的解法，由淺入深：

**第一層：使用終端機的分頁（Tabs）功能**

 macOS 的 Terminal.app 支援多分頁，你可以：
- `Command + T` 開新分頁
- `Shift + Command + →` 或 `←` 輪換分頁
- `Shift + Command + ]` 或 `[` 關閉/開啟相鄰分頁

Windows 或 Linux 也可透過系統的 `Alt + Tab` 或 `Command + Tab` 在多個終端機視窗間切換。

**第二層：使用多個實體終端機視窗**

直接開啟多個終端機視窗，利用系統的視窗管理器切換，適合需要同時對照多個檔案或命令輸出時使用。

**第三層：使用 Screen 或 Tmux（推薦）**

這才是老師最推崇的做法。Screen 與 Tmux 是命令列環境下的「虛擬視窗管理器」，它們讓你可以在**一個真實的終端機視窗**內，開啟多個虛擬終端工作階段。

以 Screen 為例，基本操作如下：

- `Ctrl + A, C`：建立新視窗
- `Ctrl + A, Ctrl + A`：在最近兩個視窗之間快速切換

為什麼推薦 Screen 或 Tmux？因為它們還具備以下實用功能：
- **超大的 scrollback buffer**：可回溯大量歷史輸出
- **分享終端機工作階段**：可讓另一個人即時觀看或共同操作
- **Detach 與 Reattach**：可將工作階段中斷連線，稍後再恢復，特別適合操作遠端伺服器時中斷連線也不會丟失工作狀態

### GVim 與命令列 Vim 的使用時機

學員提問：「身為新手程式設計師，我需要安裝 GVim 嗎？什麼時候才應該使用它？」

老師的建議是：**取決於你當下的工作情境，而非技能等級。**

GVim 適合的場景：
- 需要使用系統**剪貼簿（Clipboard）**——終端機版 Vim 對系統剪貼簿的支援有限，GVim 可以無縫存取
- 習慣用**滑鼠滾輪**或**視窗/捲軸**操作文件
- 從網頁複製文字貼入編輯器，並做一些文字加工（例如從課程報名名單中取出特定資料）

CLI 版 Vim 適合的場景：
- 幾乎所有時間都在終端機環境中工作
- 已經習慣純鍵盤操作，不依賴滑鼠

> **GVim vs CLI Vim 的抉擇，本質上是「個人偏好」與「工作情境」的問題，與你用 Vim 來寫程式、寫小說或改設定檔無關。**

### Vim 的文字處理應用：超越程式設計

有學員問到：「Vim 只能拿來寫程式嗎？對於 LaTeX 文件或一般文字處理有幫助嗎？」

老師的答案是：**Vim 是為所有文字編輯場景而生的工具，不限於程式設計。**

實際應用場景：

**LaTeX 文件寫作**
數學研究者和科學家群體中，有很多人用 Vim 來撰写 LaTeX 文件。他們會將 Vim 設定檔（`~/.vimrc`）客製化，加入專屬快捷鍵，大幅加速 LaTeX 的工作流程。

**HTML 與網頁開發**
HTML 本質上就是一種「文字格式的原始碼」。在 Vim 中你可以結合語法高亮與 Text Objects，直接操作 HTML 標籤內的內容，例如：
- `vit`：選取 HTML 標籤內的內容
- `vat`：連同標籤本身一起選取

這些功能讓你精準操控網頁結構，完全不需滑鼠。

> **簡單來說：凡是需要編輯「任何文字資料」的場景，Vim 都是強大的武器——無論那些文字最後會變成可執行檔、網頁、PDF 文件，或是純文字本身。**

## 💡 重點摘要

- **Screen 或 Tmux 是在 Vim 與終端機之間快速切換的最佳工具，同時支援 session 中斷恢復與終端分享。**
- **GVim 的核心價值在於系統剪貼簿整合與滑鼠支援；CLI Vim 則是終端機環境下的首選。**
- **Vim 不僅限於程式開發，同樣適用於 LaTeX 文件、HTML 編輯，以及任何文字加工場景。**
- **Text Objects 是操作結構成熟文件（HTML、LaTeX）最有效率的方式，能在不離開鍵盤的情況下精準選取結構化內容。**
