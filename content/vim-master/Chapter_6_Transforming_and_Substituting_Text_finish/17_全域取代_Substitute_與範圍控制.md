# Vim 全域取代：`:substitute` 與範圍控制

## 📝 課程概述

當你需要對同一個字串進行**大量替換**時，逐個 `cw` 再按 `.` 不再是最優解。本單元介紹 Vim 最接近「Find & Replace」的功能——**`:substitute`（`:s`）命令**。我們會學到如何指定替換範圍、使用 `g` flag 全域替換，以及用**自訂分隔符**解決路徑中的 `/` 字符問題。這是系統管理員修改設定檔時最重要的本職技能。

---

## 核心觀念與實作解析

### `:s` 的基本語法

```vim
:[range]s/{old}/{new}/[flags]
```

- `:s` — substitute（取代）命令本身
- `{old}` — 要尋找的目標文字
- `{new}` — 替換成的新文字
- `[flags]` — 可選的旗標

> **預設行為**：若不指定 range，`:s` **只作用於目前所在行**，且只替換該行**第一個**匹配項目。

```vim
:s/net/org/         " 只替換目前行的第一個 "net"
:s/net/org/g        " 加上 g flag → 替換該行所有 "net"
```

---

### `g` Flag：替換行內所有匹配

`g`（global）flag 讓 Vim 在一行之內**替換所有匹配**，而非僅第一個。

```text
net: 192.168.1.1
net: 10.0.0.5
```

```vim
:s/net/org/g   " 該行所有 "net" 都變成 "org"
```

> **為什麼 `g` 這麼重要？** 因為 Vim 預設 `:s` 只替換**第一個**匹配結果。對於多數真實場景（修改設定檔中的伺服器名稱、IP 位址等），你幾乎都會需要 `g` flag——這也是多數 Vim 使用者將 `g` 視為 `s` 的「預設附掛」。

---

### 範圍（Range）：精準控制作用行數

Substitute 命令最强大的延伸能力是**範圍控制**：

```vim
:1s/foo/bar/g          " 只在第 1 行執行
:1,5s/foo/bar/g        " 在第 1~5 行執行
:.,$s/foo/bar/g        " 從目前行（.）到檔案最後一行（$）
```

#### 常見特殊符號

| 符號 | 意義 |
|------|------|
| `.` | 目前所在行（current line） |
| `$` | 檔案最後一行 |
| `%` | **整個檔案**（等同 `1,$`）|

```vim
:%s/net/org/g    " 將整份檔案所有 "net" 替換為 "org"
```

> **`%` 是最常用的範圍**：修改設定檔中的主機名稱或路徑時，`:%s/old/new/g` 是幾乎所有 Vim 使用者的第一反應。

#### 範圍與 Pattern 的結合

你也可以用搜尋 pattern 作為範圍邊界：

```vim
:/global/,/local/s/net/org/g
```

這代表：從匹配 `global` 的那一行，到匹配 `local` 的那一行之間的所有行，執行替換。非常適合用在設定檔的**特定區塊**內進行修改。

---

### 替換路徑時的陷阱：自訂分隔符

**問題來了**：如果要在 Vim 中替換一個路徑，例如把 `/var/spool` 改成 `/usr/local`：

```vim
:s/\/var\/spool/\/usr\/local/g
```

你必須用 `\` 轉義每一個 `/`，不僅繁瑣，也極易出錯。

**解決方案：用其他字元當作分隔符**。Vim 允許你使用任何非英數字元的單一字元作為 pattern 分隔符：

```vim
:s#/var/spool#/usr/local#g
```

用 `#` 代替 `/` 之後，所有路徑中的 `/` 都不需要轉義，指令清晰易讀。

> **可選的分隔符**：`#`、`@`、`!`、`+`、`-`、`,` 等都可以使用。`#` 最直覺，但若替換內容本身包含 `#`，記得換用其他字元。

---

### `:set number` / `:set nonumber`：切換行號顯示

既然學到了 range（範圍），行號顯示就變得非常實用：

```vim
:set nu          " 開啟行號
:set nonu        " 關閉行號
:set nu!         " 切換（toggle）行號顯示
```

**Pattern**：`set [no]{option}` 開關選項，`set {option}!` 切換選項。

---

## 💡 重點摘要

- **`:s/old/new/g`**：`g` flag 是你的預設附掛——幾乎所有 `:s` 命令都應該加上 `g`，否則只替換每行第一個匹配結果。
- **`%` 代表整個檔案**：`:%s/old/new/g` 是修改設定檔中重複出現之值的標準做法。
- **使用自訂分隔符（如 `#`）代替 `/`**：可以徹底避免路徑中 `/` 的轉義地獄，讓 `:s` 指令保持整潔可讀。
- **範圍（Range）不只是行號**：`/pattern1/,/pattern2/` 讓你能在一對 pattern 之間的區塊內執行替換，適合大型設定檔的局部修改。
- **學會 `:set nu` 與 toggle `!` 語法**：讓行號成為你定位與確認 range 的得力工具。

---
