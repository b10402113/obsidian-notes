# Vim 插入、改寫、替換與合併：完整文字操作手冊

## 📝 課程概述

本單元是 Vim 文字操作的精華所在——我們將一次掌握 **六種進入 Insert mode 的方式**、`Replace mode` 的使用場景、`Change` 系列指令的家族成員、`Case`（大小寫）轉換指令，以及 **`J` / `gJ` 合併行**的智慧邏輯。學完這一章，你幾乎不需要離開 Normal mode 就能完成任何文字編輯任務。

---

## 核心觀念與實作解析

### 六種進入 Insert mode 的方式

你已經知道 `i` 可以從游標当前位置進入 Insert mode，但 Vim 提供了更多精準的進入點：

| 指令 | 行為 |
|------|------|
| `i` | 從游標**前方**進入 Insert mode |
| `I`（大寫） | 跳至該行**第一個非空白字元**後進入 Insert mode（等同 `0i` 或 `^i`）|
| `a` | 從游標**後方**（右側）進入 Insert mode |
| `A`（大寫） | 跳至該行**行尾**進入 Insert mode（等同 `$a`）|
| `o` | 在游標**下方**新增一行並進入 Insert mode |
| `O`（大寫） | 在游標**上方**新增一行並進入 Insert mode |

> **`I` vs `0i` 的差異**：`0` 移動到行首絕對位置，`^` 移動到該行第一個**非空白字元**。`I` 的行為等同 `^i`，因此如果行首有空白的話，`0i` 和 `I` 的游標位置並不相同。

**實用範例**：若想讓一行與上一行完全相同，無需慢慢移動到行首，**直接按 `I` 再打字是最快的方式**。同樣地，在行尾附加文字 **按 `A`** 最直覺。

---

### Count 與 Insert mode 的妙用

`Insert mode` 也可以搭配 count（計數）。當你按下一個數字再按 `i` 進入 Insert mode，完成輸入並按下 `Esc` 時，**該動作會重複 count 次**。

**經典應用場景：建立一行 80 個星號**

```text
80I*<Esc>
```

這條指令讓 Vim 在 Insert mode 中輸入一個 `*`，然後 `Esc` 時自動重複 80 次，適合用來建立分隔線或測試輸出。

**一次建立多行開頭相同的文字**

```text
5o# <Esc>
```

輸入 `# `（註解符號加空格）5 次，可用於快速建立多行註解或 YAML 設定檔。

### Replace Mode（改寫模式）

`Replace mode` 是 Insert mode 的另一種形態——**每一個輸入的字元會直接覆蓋現有字元**，而不是插入。

| 指令 | 行為 |
|------|------|
| `R`（大寫） | 進入 Replace mode，**持續**覆蓋直到按 `Esc` |
| `r`（小寫） | 覆蓋**單一字元**後自動回到 Normal mode |

**`r` 的使用時機**：只想替換一個字元，例如把 `bat` 改為 `cat`，只要把游標移到 `b` 上，輸入 `r` 再輸入 `c`，無需按 `Esc`——因為 Vim 理解你只需要替換一個字元，會自動返回 Normal mode。

**`R` 的使用時機**：需要連續覆蓋多個字元，例如把整個單字替換成另一個不同長度的單字。

> **為什麼有 `r` 和 `R` 之分？** 因為 Vim 遵循「**最小動作原則**」——如果你只需要改一個字元，就不該強迫你進入一個需要另外離開的模式。`r` 是 Vim 對「只做一件事」的優雅表達。

### Change 家族指令

`c`（Change）是 Vim 中功能最強大的指令之一，本質上是 **刪除 + 進入 Insert mode** 的複合動作：

| 指令 | 意義 |
|------|------|
| `c{motion}` | 刪除指定 motion 範圍的文字，進入 Insert mode |
| `cc` | 刪除**整行**內容，進入 Insert mode |
| `C`（大寫） | 刪除至**行尾**，進入 Insert mode（等同 `c$`）|

**`cw` — Change Word**：`cw` 刪除從游標到 word 結尾的範圍並進入 Insert mode。這在修改單字時極為實用。

```text
canine  →  游標在 c 上
cw →  輸入 cat →  cat
```

**`c$` / `C` — 改寫至行尾**：等同於「刪除整行後段 + 重新輸入」。

**`cc` — 改寫整行**：不刪除該行（保留行首空白），只清除內容後進入 Insert mode。

> **與 Register 整合**：`c`、`cc`、`C` 都服從 Vim 的 Register 機制，**被刪除的文字會自動寫入 unnamed register**，可用 `p` 貼回。若想明確指定存入哪個 Register，使用 `"a cc` 將原內容存入 `a` register。

### Case 轉換指令（大小寫）

| 指令 | 行為 |
|------|------|
| `~` | 切換游標下**一個字元**的大小寫 |
| `g~~` 或 `g~~` | 切換**整行**的大小寫 |
| `g~{motion}` | 對指定 motion 範圍做大小寫交換 |
| `gU{motion}` 或 `gUU` | 將指定範圍**轉為大寫** |
| `gu{motion}` 或 `guu` | 將指定範圍**轉為小寫** |

**為什麼 `gUU` 是兩個大寫 `U`？** 因為 Vim 的 pattern 是：`operator` + `operator` = 作用於**整行**。同樣的邏輯也出現在 `dd`（刪行）、`yy`（複製行）、`cc`（改寫行）中。

> **一個容易踩坑的地方**：`~` 是「交換」（toggle）大小寫，不是「固定」轉換。如果你有一段文字 `upper`，其中已有大寫 `E`，用 `~` 會讓 `uUPer`，這不是你要的結果。這時就應該用 `gU` 強制全大寫，或 `gu` 強制全小寫。

---

### 合併行：`J` 與 `gJ`

| 指令 | 行為 |
|------|------|
| `J`（大寫） | 合併當前行與下一行，**智慧加入空格** |
| `gJ` | 合併當前行與下一行，**不加入任何空格** |

**`J` 的智慧空格邏輯**：
- 如果目前行尾**沒有空白**， Vim 會自動加上一個空格再接下一行
- 如果目前行尾**已有句點 `.`**， Vim 會加上**兩個空格**（英文書寫排版慣例）

```text
The cat chased the mouse
Now two spaces are added.

After typing J with a period:
The cat chased the mouse.  Now two spaces are added.
```

> **何時用 `gJ` 而非 `J`？** 當你需要合併**不含空格**的資料，例如 CSV 欄位、程式碼或技術術語時，`gJ` 讓你保有完全控制權，不會被 Vim 自動干預。

---

## 💡 重點摘要

- **`I` / `A` / `o` / `O` 四個指令讓你無需先移動游標就能精準進入 Insert mode**——記住這些指令，能大幅減少按鍵次數。
- **`r` 替換單一字元後自動返回 Normal mode；`R` 進入 Replace mode 持續覆蓋。** 不要用 `R` 來做 `r` 能完成的事。
- **`c` 是「刪除 + 進入 Insert mode」的複合指令**，`cw` / `cc` / `C` 各有其適用場景，且支援 count 與 Register。
- **`gU` / `gu` 分別強制轉為全大寫或全小寫**，`~` 僅用於交換大小寫，注意兩者差異。
- **`J` 合併行時會智慧處理空格；`gJ` 則無腦拼接，選擇哪個取決於你對格式的控制需求。**
