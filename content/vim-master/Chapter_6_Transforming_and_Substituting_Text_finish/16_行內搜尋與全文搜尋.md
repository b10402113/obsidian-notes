# Vim 行內搜尋與全文搜尋：找到你想編輯的位置

## 📝 課程概述

無論你是系統管理員修改設定檔，還是開發者修復 Bug，「**快速找到目標**」是所有編輯工作的第一步。本單元分為兩個層次：**行內搜尋**（在同一行內移動到特定字元）與**檔案層級搜尋**（跨行查找字串或單字）。兩者都支援與 `d`、`c`、`y` 等 Operator 組合，形成 Vim 最強大的編輯模式之一。

---

## 核心觀念與實作解析

### 行內搜尋：`f` / `F` 與 `t` / `T`

#### `f` — Forward 往前找

`f{字元}` 將游標移動到**當前位置之後**的第一個指定字元。

```text
The quick brown fox
    ↑ 游標在 T，按 f b → 游標跳到 b
```

`f` 是 **motion**，因此可以與所有 Operator 組合：
- `df ` — 刪除到下一個空格（包含空格）
- `dtw` — 刪除到字母 `w` 之前（不包含 `w`）

#### `F` — Backward 往回找

`F{字元}` 將游標移動到**當前位置之前**的第一個指定字元，往左側搜尋。

```text
The quick brown fox
        ↑ 游標在 x，按 F q → 游標跳到 q
```

#### `t` — Till 停在目標字元之前

`t{字元}` 將游標移動到指定字元**的前一個位置**（不涵蓋目標字元本身）。`T` 則是往回找的 Till。

```text
Hello world
       ↑ 游標在 w，按 t o → 游標停在 o 之前的空白
```

> **`t` 的價值**：當你想對某個字元**之前**的所有內容做操作時，`t` 比 `f` 更精準——你不需要再按一次 `l` 避開目標字元。

### 搜尋的極速重複：`;` 與 `,`

| 指令 | 行為 |
|------|------|
| `;` | 以**相同方向**重複上一次行內搜尋 |
| `,` | 以**相反方向**重複上一次行內搜尋 |

這兩個指令讓你不需重新輸入 `f x` 就能持續往同一方向前進（`;`），或往回走（`,`）。結合 count 使用時，`3;` 相當於「往右找第 3 個」。

---

### 全文搜尋：`/` 與 `?`

`/` 是 Vim 中最核心的跨行搜尋指令。

```text
/word    →  從游標位置往下（向檔案結尾）搜尋 "word"
?word    →  往上（向檔案開頭）反向搜尋 "word"
```

| 指令 | 行為 |
|------|------|
| `n` | 往**同方向**重複上一次搜尋 |
| `N`（大寫） | 往**相反方向**重複上一次搜尋 |

**搜尋到達檔案邊界時**：Vim 會顯示 `search hit BOTTOM, continuing at TOP`（或反向的訊息），並從另一端繼續循環。

---

### 搜尋相關設定：`incsearch` 與 `hlsearch`

這兩個設定會影響你搜尋時的視覺回饋：

| 設定 | 意義 | 預設 |
|------|------|------|
| `incsearch`（`is`） | **增量搜尋**：一邊輸入一邊即時顯示目前最接近的匹配結果 | 關閉 |
| `hlsearch`（`hls`） | **搜尋結果高亮**：搜尋完後所有匹配位置保持醒目顏色 | 關閉 |

```vim
:set incsearch    " 開啟增量搜尋
:set noincsearch  " 關閉增量搜尋
:set hlsearch     " 開啟高亮
:noshlsearch      " 暫時消除高亮（直到下次搜尋）
:set nohlsearch   " 完全關閉高亮
```

> **為什麼 `hlsearch` 有兩個關閉指令？** `:nohlsearch`（或 `:nos hlsearch`）是「暫時消除當前高亮」，但設定本身仍為 `on`；`:set nohlsearch` 是「完全關閉高亮功能」。如果只想讓目前的高亮消失但日後還要用，`nos` 是對的選擇。

---

### 快速定位到特定單字：`*` 與 `#`

| 指令 | 行為 |
|------|------|
| `*` | 搜尋游標下**整個單字**（往檔案下方） |
| `#` | 搜尋游標下**整個單字**（往檔案上方） |

```text
cursor is on: the word is
按 * → 跳到下一個 "word"
按 n → 繼續往同一方向找 "word"
```

**`/` 與 `*` 的核心差異**：`/` 匹配字串的任意出現（包括 `this` 中的 `is`），而 `*` 只匹配**完整的 word**（即前後有空白或標點邊界）。如果要找的不是完整單字，用 `/` 更精確。

---

### 搜尋 motion 與 Operator 的強大組合

**搜尋指令本身就是 motion**，可以無縫接上任何 Operator：

| 組合 | 意義 |
|------|------|
| `d/something<Enter>` | 從游標位置刪除到下一個 "something" |
| `c/something<Enter>` | 從游標位置改寫到下一個 "something" |
| `y/something<Enter>` | 從游標位置複製到下一個 "something" |

**實用搜尋取代模式（重複性編輯）**：

```text
/search word<Enter>
cw replacement<Esc>
n          → 跳到下一個匹配
.          → 重複上一次的 cw 替換動作
```

這個「搜尋 → 執行指令 → `n` 跳下一個 → `.` 重複」的流程，是 Vim 中性價比最高的編輯模式之一——**用 3~4 個按鍵就能對多個位置做相同修改**。

> **`/` 也是 motion**：在 Vim 的語法框架裡，`/` 搜尋返回的是一個 **motion**，因此 `d/hello<Enter>` 語義上等同於「刪除從現在到找到 hello 之前的所有內容」。這個理解讓你能自由組合任何 Operator 與任何搜尋。

---

## 💡 重點摘要

- **`f` / `F` / `t` / `T` 是行內精準移動的基礎；`;` 重複同方向，`,` 反方向重複。**
- **`/` 與 `?` 是全文搜尋的起點；`n` / `N` 控制方向；`*` / `#` 快速找整個單字。**
- **開啟 `incsearch` 讓搜尋時即時看見匹配結果，能大幅減少輸入錯誤與來回修正的時間。**
- **`/*/#` 找的是「完整單字」；`/` 找的是「字串任意匹配」**，根據目標精準選用。
- **搜尋 motion（`/` 或 `?`）與 Operator（`d`、`c`、`y`）的組合，是 Vim 中最高效的重複編輯模式之一。**
