# 遠端開發與本地開發的差異與動機

## 📝 課程概述

本單元屬於**選讀章節**，專為硬體效能較弱的學員設計。我們將說明為何要將開發環境從本地机器遷移到 Google Cloud，以及在這個過程中 **Skaffold** 與 **Google Cloud Build** 分別扮演什麼角色。理解這套流程背後的「為什麼」，比跟著操作更重要。

---

## 核心觀念與實作解析

### 為什麼需要遠端開發？

隨著課程推進，我們會在叢集中建立越來越多的 Pod。如果你的電腦效能不足（尤其是較舊的 Windows 或 Mac 机器），可能會開始感受到明顯的延遲，甚至系統崩潰。

解決方案是：**不再在本地机器上執行 Kubernetes，而是將整個叢集搬到 Google Cloud 的虛擬機器上運行**。你的電腦只負責編輯程式碼，所有負載都交給雲端處理。

> **重要前提**：使用 Google Cloud 需要信用卡才能開通帳戶，但新帳戶會自動獲得 **$300 美元免費額度**，以本課程的用量而言，幾乎不可能用完。

---

### 本地開發架構 vs 遠端開發架構

**本地開發架構**：
- 你的電腦同時運行 Docker Desktop、單節點 Kubernetes 叢集，以及所有 Pod
- 瀏覽器透過 Ingress Nginx 存取本地節點
- 隨著 Pod 數量增加，記憶體與 CPU 消耗會直接拖慢你的机器

**遠端開發架構**：
- 你的電腦只負責運行瀏覽器與程式碼編輯器
- 整個 Kubernetes 叢集運行在 Google Cloud 的遠端 VM 上
- 瀏覽器仍然透過 Ingress Nginx 路由，但節點位於雲端

---

### Skaffold 在遠端開發中的角色

**Skaffold 是串起本地編輯與遠端叢集的關鍵工具**，它的作用分為兩種情境：

#### 情境一：修改 Sync 檔案（熱更新）

當你修改一個被列在 `sync` 區塊的檔案（例如 TypeScript 檔案）時：

1. 你在編輯器中儲存檔案
2. Skaffold 偵測到變更，直接將該檔案 **同步（sync）** 到遠端 Pod 中
3. **不需要重建 Image**，Pod 內的程式碼立即更新

```
編輯器儲存 → Skaffold 偵測 → 直接同步檔案到遠端 Pod
```

#### 情境二：修改非 Sync 檔案（完整重建 Image）

當你修改一個**不在 sync 區塊內**的檔案（例如 `package.json`）時，代表你需要重建 Image：

1. Skaffold 將**完整 source code + Dockerfile** 上傳到 Google Cloud Build
2. **Google Cloud Build** 在雲端快速建置 Image（受益於 Google 伺服器的高速網路）
3. 建好的 Image 被推送（push）到 Google Container Registry
4. Skaffold 通知 Deployment 有新 Image 可用
5. Deployment 自動重啟 Pod，使用最新 Image

```
編輯器儲存 → Skaffold 偵測 → 上傳至 Cloud Build → 雲端建置 Image → 推送至 GCR → Deployment 重啟 Pod
```

---

### 為什麼選擇 Google Cloud？

Skaffold 本身由 **Google 團隊開發**，因此與 Google Cloud 的整合程度最深、最流暢。這是課程選擇 GKE（Google Kubernetes Engine）而非 AWS EKS 或 Azure AKS 的主要原因。

---

### Google Cloud Build 的加速效益

對於依賴大量 Dependencies 的專案（例如需要下載 200-300 MB 依賴的專案），本地建置可能需要數分鐘，但透過 Google Cloud Build 可以在**數秒內**完成安裝。這是因為 Google Cloud 伺服器具備極快的網路連線，大幅縮短了下載 Dependencies 的時間。

---

## 💡 重點摘要

- **當本地机器效能不足時，將開發環境遷移到 Google Cloud 是最有效的解決方案。**
- **Skaffold 是連接本地編輯器與遠端叢集的核心橋樑，它同時處理熱更新（sync）與 Image 重建（Cloud Build）兩種情境。**
- **Google Cloud Build 的價值在於：利用雲端高速網路，大幅加速 Image 建置與 Dependencies 下載。**
- **Skaffold 由 Google 團隊開發，因此與 Google Cloud 的整合最為緊密，這是選擇 GKE 的關鍵原因。**
- **新帳戶有 $300 免費額度，課程用量極不可能超支，但仍需信用卡開通。**
