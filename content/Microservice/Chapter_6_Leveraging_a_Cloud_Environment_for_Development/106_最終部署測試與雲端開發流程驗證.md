# 最終部署測試與雲端開發流程驗證

## 📝 課程概述

經過前面幾個步驟的設定，本單元終於迎來「真實時刻」——啟動 `skaffold dev`，觀察 Skaffold 如何透過 Google Cloud Build 在雲端建置 Image，並將 Deployment、Service、Ingress 自動部署到 GKE 叢集。我們也會在瀏覽器中驗證應用程式能正常運作。

---

## 核心觀念與實作解析

### 啟動 Skaffold dev 的前置檢查

在執行 `skaffold dev` 之前，請確認以下兩件事：

1. **關閉任何舊的 Skaffold 程序**（例如 `Ctrl+C` 停止舊的 `skaffold dev` 行程）
2. **確認 kubectl Context 指向 GKE 叢集**（而非 `docker-desktop`）

確認方式：Docker Desktop → Kubernetes 選單 → 選取 GKE Context。

---

### Skaffold dev 啟動後的全流程

當你執行 `skaffold dev` 後，背後會依序發生以下事件：

1. Skaffold 偵測到 `skaffold.yaml` 中設定了 `googleCloudBuild`
2. Skaffold 將 `auth` 服務的 source code + Dockerfile 上傳到 **Google Cloud Build**
3. 你可以在 Google Cloud Console → **Tools** → **Cloud Build** → **History** 中觀看建置過程（包含 `npm install`、設定工作目錄等）
4. Cloud Build 完成後，Image 被**推送（push）**到 Google Container Registry（GCR），不再是 Docker Hub
5. Skaffold 將 Deployment、Service、Ingress 設定套用到 GKE 叢集
6. 你可以在 `skaffold dev` 的終端輸出中看到 `Console.log` 訊息——**這些訊息是從 GCP 遠端 VM 上的 Pod 輸出的**，代表應用程式已成功在雲端運行

> **很酷的視覺化驗證**：在 Cloud Build History 中點擊任一建置記錄，可以看到完整的建置日誌，其中所有輸出（安裝依賴、輸出 Image URL 等）都與本地建置時看到的幾乎相同——只不過這一切發生在 Google 的伺服器上。

---

### 在瀏覽器中驗證應用程式

1. 在瀏覽器輸入 `https://ticketing.dev/api/users/currentuser`
2. 你可能會看到 **「Your connection is not private」** 警告，這是因為 SSL 憑證尚未設定
3. 點擊 **Advanced** → **Proceed to site（不安全）** 即可繞過
4. 若看到 `{"hi":"there"}` 回應，代表應用程式**已在 GKE 叢集上正常運作**，而且瀏覽器的請求成功穿越了 GCP Load Balancer → Ingress Nginx → auth Service 的完整路徑

---

### Image 名稱管理的延續性提醒

在後續課程中，當老師回到本地環境展示程式碼時，他的 `skaffold.yaml` 會恢復成 `local: push: false` 的設定。但**你若繼續使用 GKE 遠端叢集**，必須手動將設定改回 `googleCloudBuild`，並確保：

- `skaffold.yaml` 的 `artifacts[].image` 為 `us.gcr.io/<projectId>/<service-name>`
- 各 Deployment YAML 檔案中的 `image` 欄位同步更新

同理，之後若新增 orders 或其他 Service 的 Deployment 檔案，Image URL 格式同樣要使用 GCR 格式。

---

### 停止使用後務必刪除叢集

GKE 叢集在運行期間會**持續計費**。當你：
- 完成課程後不再學習
- 或計畫休息一兩個月再回來

請務必到 Google Cloud Console 開啟該叢集，點擊 **Delete** 將其完全刪除。只要叢集存在，無論是否有流量，Google 都會收費。

---

## 💡 重點摘要

- **`skaffold dev` 啟動後，Image 建置（Cloud Build）→ 推送至 GCR → Deployment/Service/Ingress 套用到 GKE，全流程自動化完成。**
- **Cloud Build 的建置日誌可在 Console 的 Cloud Build History 中即時查看，驗證建置進度。**
- **`Console.log` 的輸出從雲端 VM 上的 Pod 而來，確認應用程式真正運行在 GCP 而非本地。**
- **繼續使用 GKE 的學員，需在每次後續課程中手動維護 `skaffold.yaml` 和 Deployment 檔案中的 Image URL（GCR 格式）。**
- **停止學習時務必刪除 GKE 叢集，否則會持續產生費用。**
