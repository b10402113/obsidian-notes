# 處理表單驗證錯誤

## 📝 課程概述

本單元將前一堂課的 Sign Up 表單加上**錯誤訊息顯示功能**。當後端 Auth Service 回傳驗證失敗時（如 Email 格式錯誤或密碼長度不符），前端須正確捕捉錯誤並以 Bootstrap Alert 樣式呈現給使用者。

## 核心觀念與實作解析

### 捕捉 API 錯誤

使用 `try/catch` 包住 Axios 請求：

```javascript
try {
  await axios.post('/api/users/signup', { email, password });
} catch (error) {
  setErrors(error.response.data.errors);
}
```

Auth Service 回傳的錯誤格式為：

```json
{
  "errors": [
    { "message": "Email must be valid", "field": "email" },
    { "message": "Password must be between 4 and 20 characters", "field": "password" }
  ]
}
```

### 錯誤陣列存入 State

```javascript
const [errors, setErrors] = useState([]);
```

`setErrors(error.response.data.errors)` 將錯誤陣列存入 state，React 偵測到 state 變化後重新 render，元資訊就能顯示在畫面上。

### 錯誤 UI 渲染

```jsx
{errors.length > 0 && (
  <div className="alert alert-danger">
    <h4>Oops...</h4>
    <ul className="my-0">
      {errors.map((err) => (
        <li key={err.message}>{err.message}</li>
      ))}
    </ul>
  </div>
)}
```

- 使用 `&&` 條件渲染：errors 有內容才顯示區塊
- `alert-danger` 套用 Bootstrap 紅色警示樣式
- `key={err.message}` 满足 React 對清單元素唯一 key 的要求

> 這裡的 `field` 欄位本可用來將錯誤訊息對應到各自 input 下方，但老師表示這會讓 React 部分過於複雜，偏離了課程對微服務的主題焦點，因此採用集中顯示的簡化做法。

---

## 💡 重點摘要

- **`try/catch` 是捕捉 API 錯誤的基本模式，錯誤物件的 `error.response.data` 才是後端實際回傳的資料內容。**
- **Auth Service 的錯誤格式固定為 `{ errors: [{ message, field }] }` 的結構，前端須據此解析。**
- **使用 `&&` 條件渲染可在 state 為空時自動隱藏整個錯誤區塊，無需額外控制變數。**

## 🔑 關鍵字
