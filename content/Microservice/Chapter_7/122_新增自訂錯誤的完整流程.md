# 122_新增自訂錯誤的完整流程

## 📝 課程概述

本單元實作一個全新的 `NotFoundError` 錯誤類別，作為「如何在系統中新增任意錯誤」的完整示範。同時也揭露了 async 函式中 `throw` 失效的問題，並透過 `express-async-errors` 套件優雅地修復這個 Express 原生的非同步缺陷。

---

## 核心觀念與實作解析

### 新增 NotFoundError 的完整步驟

**步驟一：建立錯誤類別檔案**

```typescript
// errors/not-found-error.ts
import { CustomError } from './custom-error';

export class NotFoundError extends CustomError {
  public statusCode = 404;

  constructor() {
    super('Route not found');
    Object.setPrototypeOf(this, NotFoundError.prototype);
  }

  serializeErrors() {
    return [{ message: 'Not found' }];
  }
}
```

**步驟二：在 Route Handler 中拋出**

```typescript
import { NotFoundError } from '../errors/not-found-error';

// Catch-all route：放在所有路由之後
app.all('*', async (req, res) => {
  throw new NotFoundError();
});
```

**步驟三：自動被 Middleware 捕獲**

`NotFoundError` extends `CustomError`，所以 Middleware 的 `instanceof CustomError` 判斷會自動處理它——**不需要修改 Middleware**。

### async 函式與 throw 的陷阱

當把 catch-all route 加上 `async` 關鍵字後，問題就出現了：

```typescript
// ❌ 加了 async 後，throw 的錯誤不會被 Middleware 捕獲
app.all('*', async (req, res) => {
  throw new NotFoundError();
  // 請求會在此 Hang 住，client 收不到回應
});
```

**原因**：加了 `async` 後，函式會回傳 Promise，即使 `throw` 了錯誤，Express 也無法自動捕獲它。必須手動呼叫 `next(err)`：

```typescript
// ✅ 手動呼叫 next
app.all('*', async (req, res, next) => {
  try {
    throw new NotFoundError();
  } catch (err) {
    next(err);
  }
});
```

### express-async-errors 套件：優雅的解決方案

每次 async 函式都要包 try/catch 實在太繁瑣。`express-async-errors` 這個套件會**自動修補 Express 內部對 async 錯誤的處理機制**，讓 async 函式中的 `throw` 表現得跟同步函式一樣：

**安裝：**
```bash
npm install express-async-errors
```

**使用：**
```typescript
// 在 index.ts 中，express 之後立即引入
import express from 'express';
import 'express-async-errors'; // 只需要引入，不需呼叫任何函式

// 現在可以這樣寫，完全不需要 try/catch
app.all('*', async (req, res) => {
  throw new NotFoundError(); // ✅ 會自動被 Middleware 捕獲
});
```

> **原理**：`express-async-errors` 會在內部對 `Router` 原型注入一個 `async` wrapper，讓 Express 能自動監聽 Promise rejection，並將其轉發給 Error Handling Middleware。

---

## 💡 重點摘要

- **新增任何 Error 類別的流程：建立 `extends CustomError` 的 class → 在 Route Handler 中 `throw` → Middleware 自動處理，無需修改。**
- **在 async Route Handler 中，`throw` 的錯誤不會被 Express 自動捕獲，這是 async/await 與 Express 整合的已知問題。**
- **`express-async-errors` 是目前最優雅的解法——只需要引入一次，所有 async 函式中的 `throw` 都會正確送達 Middleware。**
