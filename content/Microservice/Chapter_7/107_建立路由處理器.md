# 107_建立路由處理器

## 📝 課程概述

本單元是認證服務（auth service）開發的起點。我們將建立四個路由處理器（`signin`、`signout`、`signup`、`currentUser`），並採用統一的檔案結構與命名慣例。這套 convention 將貫穿未來所有 Service 的實作，確保程式碼高度一致且易於維護。

---

## 核心觀念與實作解析

### 為什麼要將 Route Handler 拆分成獨立檔案？

在 Express 應用中，常見的做法是直接在 `index.ts` 裡一口氣定義所有路由。然而當專案規模擴大後，這種做法會讓 `index.ts` 變得極度肥腫，難以維護。

老師選擇的策略是：**每個 Route Handler 各自擁有獨立檔案**，好處是：

- 責任分明，單一檔案只做一件事
- 未來擴充新路由時不會動到既有檔案
- 多人協作時的 merge conflict 最小化

### Router 物件的建立方式

每個 Handler 檔案的結構都遵循同一套模板：

```typescript
import express, { Request, Response } from 'express';
const router = express.Router();

router.get('/api/users/currentuser', (req: Request, res: Response) => {
  res.send({ message: 'Hi there' });
});

export { router as currentUserRouter };
```

這裡有幾個值得注意的細節：

1. **`express.Router()`**：建立一個 Router 物件，用來關聯多個路由規則
2. **重新匯出（rename export）**：底部 `export { router as currentUserRouter }` 的目的是避免多個 Router 檔案匯出時都叫 `router`，造成名稱衝突
3. **Type Annotations**：`req` 與 `res` 都需要明確標註 `Request` 與 `Response` 型別，避免 TypeScript 推導失敗

### 在 index.ts 中組裝

回到 `index.ts`，我們刪除原本內嵌的 Handler，並以 `app.use()` 掛載各 Router：

```typescript
import { currentUserRouter } from './routes/current-user';
import { signinRouter } from './routes/signin';
import { signoutRouter } from './routes/signout';
import { signupRouter } from './routes/signup';

app.use(currentUserRouter);
app.use(signinRouter);
app.use(signoutRouter);
app.use(signupRouter);
```

> **留意**：這裡 `app.use()` 的順序很重要——Express 會依序匹配路由，雖然多個 Router 各自處理不同路徑，但維持一致的掛載順序是好的團隊慣例。

---

## 💡 重點摘要

- **每個 Route Handler 獨立一個檔案，是為了未來擴充 Service 時保持一致的程式碼結構。**
- **`express.Router()` 讓我們可以在不同檔案中定義路由，再統一掛載到主應用程式。**
- **重新匯出時 rename（`as xxxRouter`）是避免多個 Router 模組之間命名衝突的必要手法。**
- **TypeScript 的 `Request` 與 `Response` 型別註解能預防許多型別相關的編譯錯誤。**
