# Why You Should Use GitHub — 使用 GitHub 的價值

## 📝 課程概述

本節從更廣的視野說明為什麼要學習 GitHub，而不僅僅是「可以備份程式碼」。GitHub 是全球最大的開源專案聚集地，是求職時的技術履歷，也是開發者之間建立人脈與追蹤技術動態的平台。對於任何想在軟體開發行業發展的人來說，GitHub 都是必備技能。

## 核心觀念與實作解析

### GitHub 的核心價值

**1. 雲端代管與備份**

將本地 Git Repo 推送到 GitHub 後，你的程式碼就有了雲端備份。即使本地硬碟損壞、筆電遺失，只要能連上網路，就能從 GitHub 將程式碼完整拉回來。

> **但這並不是 GitHub 最大的價值**——雲端備份有太多替代方案（iCloud、Google Drive 等）。GitHub 真正的強大之處在於協作與開源生态。

**2. 多人協作**

GitHub 的核心功能是讓多個開發者能夠：
- 各自 Clone 同一個 Repo 到自己的機器
- 在各自的分離分支上開發功能
- 將變更推送到 GitHub
- 彼此檢視、討論、Merge 程式碼

熱門專案如 React 有 1,500+ 貢獻者，TensorFlow 有近 3,000 名貢獻者——這種規模的協作完全依賴 GitHub 這類平台。

**3. 開源專案的家園**

GitHub 是全球最大的開源專案代管平台。你可以：
- 查看熱門專案的源碼（React、TensorFlow、Vue、Next.js 等）
- 為開源專案提交 Bug 修復或新功能
- 參與自己喜歡的專案開發

**4. 技術履歷（GitHub Profile）**

雇主在面試開發者時，經常會查看 GitHub Profile。GitHub Profile 展示了：
- 你的貢獻歷史（Contribution Graph）
- 你曾參與的專案（Repositories）
- 你的程式碼風格與品質

一個活跃的 GitHub 帳號，比任何紙本履歷都更能證明你的實作能力。

**5. 保持技術敏感度**

關注熱門開源專案的通知（Notifications），可以：
- 提前知道下一個大版本的功能更新
- 參與技術討論，了解業界最新趨勢
- 遇到有興趣的維護者，主動建立技術交流的機會

### GitHub 的定價模型

| 方案 | 價格 | 主要特性 |
|------|------|----------|
| Free（免費）| 免費 | 公開/私人 Repo 無限、協作者無限、基本 CI/CD |
| Team | 付費/人 | 更多的 GitHub Actions 時間、組織管理功能 |
| Enterprise | 付費/人 | SSO、SAML、進階審計功能 |

對於個人開發者和小型團隊，**免費方案已經提供了所有核心功能**。付費方案主要是企業級功能。

## 💡 重點摘要

- GitHub 不只是「雲端硬碟」，它的核心價值是**多人協作**與**開源生態**
- 參與開源專案是很好的學習方式，也是求職時的加分項目
- GitHub Profile 是技術人的**線上履歷**，展示了你的程式碼風格與貢獻能力
- GitHub 的免費方案功能完整，足以應付絕大多數個人與小型團隊需求
- 追蹤熱門開源專案可以幫助你保持技術敏感度，了解業界最新動態
